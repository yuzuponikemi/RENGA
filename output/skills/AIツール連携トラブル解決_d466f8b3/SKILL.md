# AIツール連携トラブル解決

**説明**: Ollama等のAIツールの設定・接続エラーを解決する
**ステータス**: ✅ 公開中 | **ソース数**: 4 件 | **ユニークユーザー**: 4 人 | **カテゴリ**: トラブルシューティング支援

---

## テンプレートプロンプト

```
私は{ツール名}の{問題内容}で困っています。{環境情報}を使用しています。{試したこと}を試しましたが解決しません。{目標}ために、{具体的な質問}を教えてください。{追加要件}
```

## 変数一覧

- `{{ツール名}}`: 使用しているAIツール名（例: Ollama, Claude Code, Neo4j Desktop）
- `{{問題内容}}`: 発生している具体的な問題（例: モデルが利用できない、接続エラーになる）
- `{{環境情報}}`: OSやバージョンなどの環境情報（例: Windows 11, macOS Sonoma）
- `{{試したこと}}`: ユーザーが既に試した対処法（例: モデルのpull, 設定変更）
- `{{目標}}`: 最終的に達成したいこと（例: ローカルモデルで動作させる、コスト削減）
- `{{具体的な質問}}`: 知りたい具体的な設定手順や設定値（例: 環境変数の設定方法、ベースURLの形式）
- `{{追加要件}}`: その他の条件や要望（例: オフラインでも使いたい、人間が追跡可能なログが欲しい）

## 活用シーン

- ローカルLLMを外部ツールのバックエンドとして設定する方法を質問する
- データベース接続ツールでエラーが出る原因を調べる
- ローカルLLMとクラウドLLMのハイブリッド運用方法を相談する

## トリガーフレーズ

- `接続エラーになる`
- `設定方法を教えて`
- `Ollamaをバックエンドにしたい`
- `モデルが使えない`
- `オフラインで使いたい`

## 具体インスタンス

### インスタンス 1: Claude Code×Ollama設定

> 元プロンプト: *The user wants to know how to use Ollama as a backend for Claude Code itself. Specifically:  1. Can Claude Code use Olla*

**変数の値**
  - `ツール名`: Ollama
  - `別のツール名`: Claude Code
  - `OS名`: macOS Sonoma
  - `クラウドモデル名`: Claude Opus

**実際のプロンプト**
```
私は{ツール名}を{別のツール名}のバックエンドとして使う方法で困っています。{OS名}を使用しています。{ツール名}でモデルをpullしてリスト表示できることは確認しました。{別のツール名}の推論には{クラウドモデル名}を使い、簡単なタスクだけ{ツール名}に委譲してトークンコストを節約したいです。環境変数の設定方法、ベースURLの形式、モデル名の指定形式を教えてください。オフライン時に完全に{ツール名}に切り替える方法も知りたいです。
```

