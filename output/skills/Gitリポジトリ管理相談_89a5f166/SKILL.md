## テンプレートプロンプト

```
この{リポジトリ名}リポジトリだけど、git的にはどう管理すればいいかな　すでに{フォーク状態}してるから普通にPushすると自分のものとして開発できる？　{注意点}ので迷惑かけたくない

なるほどね　今後何かソフトウェアリポジトリをこの{リポジトリ名}に開発してもらって、それを単体リポジトリとして私の{ホスティングサービス}で確認したいと思った時はどうすればいいでしょうか

OKです　そしたら簡便化のため　今後は　{個人用フォルダパス} みたいなフォルダを作って、そこは全部アクセス可能にしとこうかな　そこに新しいリポジトリ作って作業してもらう感じでいきましょうか　たとえば私がこれまで開発してきたプロジェクトも、それで安全かな　もしくは、{CLIツール}を使えるように渡してるんだから、{ホスティングサービス}上の資産全消しとかも今可能になってます？

了解です　今{新規フォルダ名}というフォルダを作成して、{ホスティングサービス}リポジトリにして、{認証トークン}のアクセス先もそこだけに限定しました　この{リポジトリ名}に渡すアクセスもこのパスだけに限定してもらえますか
```

## 変数一覧

- `{{リポジトリ名}}`: 管理対象のリポジトリ名
- `{{フォーク状態}}`: フォーク済みかどうか（例: フォークしてる）
- `{{注意点}}`: 気をつけたい点（例: OSSなので）
- `{{ホスティングサービス}}`: Gitリポジトリのホスティングサービス名（例: github）
- `{{個人用フォルダパス}}`: 個人用フォルダのパス（例: personal/yourname）
- `{{CLIツール}}`: 使用するCLIツール名（例: gh cli）
- `{{新規フォルダ名}}`: 新しく作成するフォルダ名（例: yourname-pg）
- `{{認証トークン}}`: アクセス制限に使うトークン（例: token）
