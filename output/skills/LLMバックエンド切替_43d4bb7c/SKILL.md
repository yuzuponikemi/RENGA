# LLMバックエンド切替

**説明**: メインのLLMとサブのLLMを状況に応じて切り替えるスキル
**ステータス**: 🔒 審査中 | **ソース数**: 4 件 | **ユニークユーザー**: 2 人 | **カテゴリ**: LLMバックエンド切替

---

## テンプレートプロンプト

```
{メインLLM}を普段使いしているが、{制限条件}が発生した場合に備えて、{サブLLM}への切り替え方法を知りたい。特に、{トリガー方法}で動的にバックエンドを切り替えられるようにしたい。{追加要件}があれば教えてください。
```

## 変数一覧

- `{{メインLLM}}`: 普段使用しているLLMサービス（例: 高性能LLM、軽量LLM）
- `{{サブLLM}}`: 切り替え先のLLM（例: ローカルLLM、別のクラウドLLM）
- `{{制限条件}}`: 切り替えを検討する状況（例: リミット超過、ネットワーク断、トークン節約）
- `{{トリガー方法}}`: 切り替えの起動方法（例: コマンド、自動検知、ボタン）
- `{{追加要件}}`: その他要望（例: トークン節約、オフライン対応、特定ツールとの連携）

## 活用シーン

- APIリミット超過時にローカルLLMへ自動フォールバック
- ネットワークがない環境で完全オフラインLLMに切り替え
- 単純タスクは軽量LLM、複雑タスクは高性能LLMに振り分け

## トリガーフレーズ

- `バックエンド切り替え方法を教えて`
- `LLMを切り替えたい`
- `リミット時のフォールバック設定`
- `オフラインでも使えるようにしたい`
- `トークン節約のために軽量モデルに切り替え`

## 具体インスタンス

### インスタンス 1: Claude→Ollama切替

> 元プロンプト: *claude codeそれ自体もバックエンドをollamaで実行できると知りました　例えば、メインの推論はclaude opusのままで、単純なタスクはollamaに降ってトークン節約とか、ネットワークがない時は完全にollamaに切り替え*

**変数の値**
  - `メインLLM`: 高性能LLM
  - `サブLLM`: ローカルLLM
  - `制限条件`: ネットワークがない時、トークン節約したい時
  - `トリガー方法`: 自動振り分け（単純タスクはローカルLLM）
  - `追加要件`: 開発ツールのバックエンドとしてローカルLLMを設定

**実際のプロンプト**
```
あなたの使用している開発ツールそれ自体もバックエンドをローカルLLMで実行できると知りました。例えば、メインの推論は高性能LLMのままで、単純なタスクはローカルLLMに降ってトークン節約とか、ネットワークがない時は完全にローカルLLMに切り替えるとか、やり方を教えてください。
```
