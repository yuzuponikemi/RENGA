# 🔷 スキル定義: Neo4jトラブルシューティング

**説明**: Neo4jの起動・接続・操作に関する問題解決を依頼するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **カテゴリ**: Neo4jトラブルシューティング

---

## テンプレートプロンプト（抽象）

```
{Neo4j環境}で{問題の状況}が発生しています。{具体的な症状}を確認して、{希望する対応}を実施してください。
```

## 変数一覧

- `{{Neo4j環境}}`: 使用しているNeo4jの環境（例: Docker、Neo4j Desktop、サーバー）
- `{{問題の状況}}`: 問題が発生したタイミングや操作（例: 起動時、接続時、マージ後）
- `{{具体的な症状}}`: エラーメッセージや異常な動作の詳細（例: 接続設定でエラー、動作しない）
- `{{希望する対応}}`: 依頼者が希望する解決策や確認内容（例: 起動確認、再起動、デバッグ）

## 活用シーン

- Neo4j Desktopで接続エラーが発生した際のトラブルシューティング
- Docker上のNeo4jが起動しない問題の解決依頼
- コードマージ後にNeo4jが動作しなくなった場合の調査依頼

# 🔶 具体インスタンス

### インスタンス 1: Docker起動確認

> 元プロンプト: *neo4jは起動できますか？Docker本体は起動しておきました*

**変数の値**
  - `Neo4j環境`: Docker
  - `問題の状況`: 起動時
  - `具体的な症状`: Docker本体は起動済みだがNeo4jが起動できるか不明
  - `希望する対応`: 起動の可否確認と起動方法の提示

**実際のプロンプト**
```
{Neo4j環境}でNeo4j環境が起動できますか？{Neo4j環境}本体は起動しておきました。起動の可否を確認して、起動方法を教えてください。
```
