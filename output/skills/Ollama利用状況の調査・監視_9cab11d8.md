# 🔷 スキル定義: Ollama利用監視テンプレート

**説明**: Ollamaのログ・性能・設定を調査・監視するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 4 件 | **カテゴリ**: Ollama利用状況の調査・監視

---

## テンプレートプロンプト（抽象）

```
私は{ツール名}を使って{モデル群}を運用しています。{調査目的}のために、{具体的な依頼内容}を実施してください。特に{注目ポイント}に注意し、結果を{出力形式}で報告してください。
```

## 変数一覧

- `{{ツール名}}`: 使用しているツール名（例: Ollama, MLX）
- `{{モデル群}}`: 対象モデルの説明（例: 複数のLLMモデル, 特定のモデル名）
- `{{調査目的}}`: 調査の目的（例: 性能測定, ログトレース, 設定確認）
- `{{具体的な依頼内容}}`: 具体的に実行してほしいタスク（例: ベンチマークコード作成, プロキシ経由確認, コンテキスト設定検証）
- `{{注目ポイント}}`: 特に注目すべき点（例: コンテキスト設定, プロキシ経由, レポート品質）
- `{{出力形式}}`: 希望する出力形式（例: レポート, コード, 設定確認結果）

## 活用シーン

- 運用中のモデルのベンチマークを依頼する
- プロキシ経由のログトレースを確認する
- モデルのコンテキスト設定が適切か検証する

# 🔶 具体インスタンス

### インスタンス 1: モデル性能ベンチマーク

> 元プロンプト: *今ollamaでたくさんモデル持っているのですが、このPCでそれぞれどういう性能が出るのかを調べたいです　ベンチマーク用のコードを書いて、結果をリポートしてくれますか*

**変数の値**
  - `ツール名`: Ollama
  - `モデル群`: 複数のLLMモデル
  - `調査目的`: 性能測定
  - `具体的な依頼内容`: ベンチマーク用のコードを書いて、結果をリポート
  - `注目ポイント`: 各モデルの推論速度とメモリ使用量
  - `出力形式`: 表形式のレポート

**実際のプロンプト**
```
私は{ツール名}を使って複数のLLMモデルを運用しています。それぞれの性能を調べるために、ベンチマーク用のコードを書いて、結果をリポートしてくれますか。特に各モデルの推論速度とメモリ使用量に注意し、結果を表形式のレポートで報告してください。
```

### インスタンス 2: プロキシ経由確認

> 元プロンプト: *OK podcast-archiveがしようするollamaなんですけど、ちゃんとOllamaProxサーバーを経由しているでしょうか？どのモデルをどう使っているのかトレースしたいので*

**変数の値**
  - `ツール名`: Ollama
  - `モデル群`: {プロジェクト名}用のモデル
  - `調査目的`: ログトレース
  - `具体的な依頼内容`: どのモデルをどう使っているのかトレース
  - `注目ポイント`: プロキシ経由の有無とモデル使用履歴
  - `出力形式`: トレースログ

**実際のプロンプト**
```
私は{ツール名}を使って{プロジェクト名}用のモデルを運用しています。ちゃんと{プロキシサーバー名}を経由しているかを確認するために、どのモデルをどう使っているのかトレースしたいので、ログ取得と経路確認の手順を実施してください。特にプロキシ経由の有無とモデル使用履歴に注意し、結果をトレースログの形式で報告してください。
```
