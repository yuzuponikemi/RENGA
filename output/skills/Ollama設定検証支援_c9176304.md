# 🔷 スキル定義: Ollama設定検証支援

**説明**: Ollamaのモデル設定・接続検証とドキュメント作成を支援する
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **カテゴリ**: Ollama設定検証支援

---

## テンプレートプロンプト（抽象）

```
あなたはOllamaの設定検証とドキュメント作成の専門家です。以下の指示に従って作業を進めてください。

【検証対象】
- 使用するOllamaモデル: {model_name}
- 検証する設定項目: {config_items}
- 対象プラットフォーム/ツール: {target_tool}

【作業手順】
1. まず、{model_name} が利用可能かどうかを `ollama list` で確認してください。利用不可の場合は `ollama pull {model_name}` で取得してください。
2. {target_tool} でOllamaをバックエンドとして使用するための設定手順を、以下の観点で検証・説明してください：
   - {config_items}
3. 各ステップの中間出力を人間が追跡可能な形でドキュメント化してください。
4. 制限事項や注意点があれば明記してください。

【出力形式】
- 各検証ステップごとに、実行内容・結果・考察を記録したドキュメントをMarkdown形式で作成してください。
- 環境変数や設定ファイルの具体例を含めてください。
```

## 変数一覧

- `{{model_name}}`: 使用するOllamaモデル名（例: llama3.2, mistral, codellama）
- `{{config_items}}`: 検証する設定項目のリスト（例: 環境変数, ベースURL, モデル名フォーマット, ハイブリッド設定）
- `{{target_tool}}`: Ollamaを連携させる対象ツール（例: 特定のコードエディタ, カスタムスクリプト, 汎用環境）

## 活用シーン

- Ollamaを特定のコードエディタのバックエンドとして設定する手順を検証・文書化する
- ローカルLLMとクラウドLLMのハイブリッド構成を検証する
- オフライン環境でOllamaのみを使用する設定を検証する
