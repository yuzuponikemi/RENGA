## テンプレートプロンプト

```
{URL} に対して、{作業内容} を実施してください。具体的には {詳細指示} を行います。
```

## 変数一覧

- `{{URL}}`: 操作対象のWebページやリポジトリのURL（例: https://example.com/page）
- `{{作業内容}}`: 行う作業の種類（例: 確認、登録、Push、フォーク）
- `{{詳細指示}}`: 作業の具体的な手順や注意点（例: ブログの有無を確認、色の違いをチェック）

## 具体インスタンス

### インスタンス 1: ブログ消失確認

> 元プロンプト: *サイトに行くと、ブログが無くなっちゃてますね　あと　aboutの内容は　Homeに表示しとけばいいと思おう　あと　substackのフォームだけ色が違うので浮いてますね　{URL}　をplaywriteで実際にみて確認してくれますか*

**変数の値**
  - `URL`: https://example.com/blog/
  - `作業内容`: ブログの有無とデザイン確認
  - `詳細指示`: Playwright で実際にページを開き、ブログがなくなっているか、aboutの内容をHomeに表示すべきか、Substackのフォームの色が浮いていないかをチェック

**実際のプロンプト**
```
{URL} に対して、ブログの有無とデザイン確認を実施してください。具体的には Playwright で実際にページを開き、ブログがなくなっているか、aboutの内容をHomeに表示すべきか、Substackのフォームの色が浮いていないかをチェックします。
```

### インスタンス 2: リポジトリ登録Push

> 元プロンプト: *{URL} に作成しましたので　登録とPushしてください　ghはいれといて*

**変数の値**
  - `URL`: https://github.com/{ユーザー名}/{リポジトリ名}
  - `作業内容`: リポジトリの登録とPush
  - `詳細指示`: GitHub CLI (gh) がインストール済みの環境で、リポジトリをクローンまたは初期化し、変更をコミットしてリモートにPush

**実際のプロンプト**
```
{URL} に対して、リポジトリの登録とPushを実施してください。具体的には GitHub CLI (gh) がインストール済みの環境で、リポジトリをクローンまたは初期化し、変更をコミットしてリモートにPushします。
```

  - `詳細指示`: このリポジトリを自分のGitHubアカウントにフォークし、その後ローカルにクローンして開発を進める手順を説明

**実際のプロンプト**
```
{URL} に対して、リポジトリのフォークと開発方法の調査を実施してください。具体的には このリポジトリを自分のGitHubアカウントにフォークし、その後ローカルにクローンして開発を進める手順を説明します。
```
