# 🔷 スキル定義: URL確認・操作依頼

**説明**: 指定URLの状態確認や操作をAIに依頼するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 4 件 | **カテゴリ**: URL確認・操作依頼

---

## テンプレートプロンプト（抽象）

```
{URL} にアクセスして、{確認・操作内容} を実行してください。必要に応じて{追加指示}も行ってください。
```

## 変数一覧

- `{{URL}}`: 確認・操作対象のWebページURL（例: https://example.com/blog/）
- `{{確認・操作内容}}`: 具体的な確認項目や操作内容（例: ブログの有無を確認、フォームの色を確認）
- `{{追加指示}}`: 補足の指示や条件（例: Playwrightで実際に表示確認、GitHubにPush）

## 活用シーン

- デプロイ済みWebサイトの表示確認と修正提案
- GitHubリポジトリの状態確認と操作依頼
- 特定ページのUI要素チェックと改善提案

# 🔶 具体インスタンス

### インスタンス 1: ブログ消失確認

> 元プロンプト: *サイトに行くと、ブログが無くなっちゃてますね　あと　aboutの内容は　Homeに表示しとけばいいと思おう　あと　substackのフォームだけ色が違うので浮いてますね　{URL}　をplaywriteで実際にみて確認してくれますか*

**変数の値**
  - `URL`: {URL}
  - `確認・操作内容`: ブログが無くなっているか確認、aboutの内容をHomeに表示する提案、substackのフォームの色が浮いていないか確認
  - `追加指示`: Playwrightで実際に表示確認

**実際のプロンプト**
```
{URL} にアクセスして、ブログが無くなっているか確認してください。あと、aboutの内容はHomeに表示しとけばいいと思います。さらに、substackのフォームだけ色が違うので浮いているか確認してください。Playwrightで実際に表示確認してください。
```

### インスタンス 2: リポジトリ追加開発

> 元プロンプト: *{URL} このリポジトリを自分のgithubに加えて開発して行ったりしたいけどどうすれば良い？*

**変数の値**
  - `URL`: {URL}
  - `確認・操作内容`: リポジトリを自分のGitHubに追加して開発する方法を調査
  - `追加指示`: 具体的な手順を説明

**実際のプロンプト**
```
{URL} このリポジトリにアクセスして、自分のGitHubに追加して開発していく方法を教えてください。どうすれば良いか具体的な手順を説明してください。
```

### インスタンス 3: デプロイ確認と質問生成

> 元プロンプト: *{URL}　に今実際デプロイ済みです　私への質問リストを生成しながら、記載を進めていってください*

**変数の値**
  - `URL`: {URL}
  - `確認・操作内容`: デプロイ済みの状態確認、質問リスト生成
  - `追加指示`: 質問リストを生成しながら記載を進める

**実際のプロンプト**
```
{URL} にアクセスして、今実際にデプロイ済みの状態を確認してください。私への質問リストを生成しながら、記載を進めていってください。
```
