# 🔷 スキル定義: エラーメッセージ要約と修正

**説明**: エラーメッセージを要約し、修正方法を提案するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 10 件 | **カテゴリ**: エラーメッセージ要約

---

## テンプレートプロンプト（抽象）

```
以下のエラーメッセージを要約し、修正方法を提案してください。

エラーメッセージ: {error_message}

コンテキスト: {context}

出力形式:
- エラー概要: {error_summary}
- 原因: {cause}
- 修正方法: {fix_method}
```

## 変数一覧

- `{{error_message}}`: 発生したエラーメッセージ全文（例: 無料試用版サブスクリプションは、Flex 従量課金 ではサポートされていません）
- `{{context}}`: エラーが発生した状況や操作内容（例: 指定したクラウドサービスでのリソース作成時）
- `{{error_summary}}`: エラーの簡潔な要約（例: 無料試用版サブスクリプションではFlex従量課金が利用不可）
- `{{cause}}`: エラーの原因（例: サブスクリプションの種類と料金プランの不一致）
- `{{fix_method}}`: 具体的な修正手順（例: 有料サブスクリプションに変更するか、別の料金プランを選択）

## 活用シーン

- クラウドサービスのデプロイエラーを解析・修正する
- アプリケーションのビルドエラーを要約してチームに共有する
- ユーザーから報告されたエラーを分析し、サポート対応する

> 元プロンプト: *指定したサブスクリプション 無料試用版サブスクリプションは、App Service ではサポートされていません。別のサブスクリプションを選択してください。 やはりだめかも　確認および作成をおすとこのエラーになりますね*

**変数の値**
  - `error_message`: 無料試用版サブスクリプションは、Flex 従量課金 ではサポートされていません。別のサブスクリプションを選択してください。

  - `cause`: サブスクリプションの種類（無料試用版）と料金プラン（Flex従量課金）の不一致
  - `fix_method`: 有料サブスクリプションに変更するか、App Service の料金プランを無料試用版でサポートされているものに変更

コンテキスト: 指定したサブスクリプションで App Service を作成しようとした際に発生

### インスタンス 2: Gitプッシュエラー対応

> 元プロンプト: *全部Pushしてください*

**変数の値**
  - `error_message`: リモートリポジトリへのプッシュが拒否されました。ブランチの状態が一致していません。
  - `context`: ローカルでコミット後、git push を実行した際に発生
  - `error_summary`: リモートとローカルのブランチ状態不一致によりプッシュ拒否
  - `cause`: リモートブランチにローカルにない変更がある
  - `fix_method`: git pull でリモートの変更を取り込んでから再度プッシュする

**実際のプロンプト**
```
以下のエラーメッセージを要約し、修正方法を提案してください。

エラーメッセージ: リモートリポジトリへのプッシュが拒否されました。ブランチの状態が一致していません。

コンテキスト: ローカルでコミット後、git push を実行した際に発生

出力形式:
- エラー概要: リモートとローカルのブランチ状態不一致によりプッシュ拒否
- 原因: リモートブランチにローカルにない変更がある
- 修正方法: git pull でリモートの変更を取り込んでから再度プッシュする
```
