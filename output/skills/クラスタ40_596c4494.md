# 🔷 スキル定義: 処理実行と確認依頼

**説明**: 特定の処理を実行し、結果を確認・デバッグするスキル
**ステータス**: ✅ 公開中 | **ソース数**: 7 件 | **カテゴリ**: クラスタ40

---

## テンプレートプロンプト（抽象）

```
{処理内容}を{実行方法}してみて、{確認項目}を確認して
```

## 変数一覧

- `{{処理内容}}`: 実行したい具体的な作業や処理（例: 指定したデータの生成、既存コンテンツの再処理）
- `{{実行方法}}`: 実行の方式や条件（例: 特定のオプション付きで、再度実行）
- `{{確認項目}}`: 確認したい結果やメタデータ（例: 出力内容の状態、誤りがないか）

## 活用シーン

- 新しいデータ処理パイプラインの動作確認
- 既存コンテンツの品質チェックと修正
- デバッグ目的での再実行と結果検証

### インスタンス 1: トリプル生成と確認

> 元プロンプト: *1エピソードを話者分離付きでトリプル生成してみて、どんなメタデータになってるか確認して*

**変数の値**
  - `処理内容`: 指定したエピソードのトリプル生成
  - `実行方法`: 話者分離付きで
  - `確認項目`: どんなメタデータになってるか

**実際のプロンプト**
```
指定したエピソードを話者分離付きでトリプル生成してみて、どんなメタデータになってるか確認して
```
