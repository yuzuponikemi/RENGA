# 🔷 スキル定義: コードベース探索・文書化

**説明**: プロジェクト構造を徹底調査し文書化するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 9 件 | **カテゴリ**: コードベース探索・文書化

---

## テンプレートプロンプト（抽象）

```
Explore the project at {project_path} thoroughly. I need to understand:
1. The overall project structure (all directories and key files)
2. What language/framework is used
3. Any existing CLI setup, configuration patterns, or data processing pipelines
4. The purpose of the project (read README, any docs, main entry points)
5. Any existing models, schemas, or data structures
6. Dependencies ({dependency_files})
7. Test patterns if any exist

Be very thorough - list all directories, key files, and their purposes.

Additionally, please provide:
- Complete file tree (every file) excluding {exclude_dirs}
- For each source file in {source_dir}, list the public classes, functions, and their one-line docstrings
- The full contents of {config_files}
- All CLI commands available (read {cli_file})
- All documentation files (README, .md files in root or docs/)
- Key design patterns used
- Data flow: what each stage receives as input, produces as output, and saves to disk
```

## 変数一覧

- `{{project_path}}`: 探索対象のプロジェクトの絶対パス
- `{{dependency_files}}`: 依存関係ファイル名（カンマ区切り）
- `{{exclude_dirs}}`: 除外するディレクトリ名（カンマ区切り）
- `{{source_dir}}`: 主要ソースコードのディレクトリ
- `{{config_files}}`: 設定ファイルのパス（カンマ区切り）
- `{{cli_file}}`: CLIエントリポイントファイルのパス

## 活用シーン

- 新規参画メンバーがプロジェクト全体を理解するための初期調査
- リファクタリング前に既存コードベースの全体像を把握する
- ドキュメント作成・更新のためにプロジェクト構造を網羅的に収集する
