# サイト記事構造分析

**説明**: 指定サイトの記事構造・リンク・RSSを分析するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **ユニークユーザー**: 1 人 | **カテゴリ**: サイト記事構造分析

---

## テンプレートプロンプト

```
以下のURLの構造を調べてください。WebFetchで実際のHTMLを取得して分析してください。

1. {サイトURL} — {ページ種別}
2. 最新記事を{記事本数}本（{記事取得方法}）

調べたいこと:
- 記事の構造（{分析項目1}）
- 記事内でリンクされている外部記事の扱い（{分析項目2}）
- {追加分析項目}
- 1記事の文字数の目安
- {ページ構造情報}

{出力形式}で報告してください。
```

## 変数一覧

- `{{サイトURL}}`: 分析対象のWebサイトURL（例: https://example.com/）
- `{{ページ種別}}`: 最初に取得するページの種類（例: トップページ）
- `{{記事本数}}`: 分析する記事の本数（例: 最新5件）
- `{{記事取得方法}}`: 記事の見つけ方（例: トップページからURLを見つけてフェッチ）
- `{{分析項目1}}`: 記事構造の詳細（例: タイトル、本文、外部リンクの数と種類）
- `{{分析項目2}}`: 外部リンクの分析内容（例: どのくらいの数、どんなサイト）
- `{{追加分析項目}}`: 追加で調べる項目（例: RSSフィードがあるか）
- `{{ページ構造情報}}`: ページ構造に関する情報（例: ページネーションや過去記事へのアクセス方法）
- `{{出力形式}}`: 報告の形式と文字数制限（例: 100語以内で簡潔に報告）

## 活用シーン

- 新しいブログサイトの記事構造を調査するとき
- 競合サイトのリンク戦略を分析するとき
- RSSフィードの有無や記事更新頻度を確認するとき

## 具体インスタンス

### インスタンス 1: Pluralistic詳細分析

> 元プロンプト: *指定のサイトを詳しく調べてください。  1. RSSフィード {サイトURL}/feed/ をフェッチして、最新5件の記事タイトル・URL・日付を列挙 2. 最新記事を1本フルでフェッチして以下を分析:    - 本文の構造（見出し、段落数*

**変数の値**
  - `サイトURL`: https://pluralistic.net/
  - `ページ種別`: RSSフィード
  - `記事本数`: 最新5件
  - `記事取得方法`: RSSフィードをフェッチ
  - `分析項目1`: 本文の構造（見出し、段落数）、外部リンクを全部列挙（アンカーテキスト + URL）
  - `分析項目2`: リンクをカテゴリ分類: 「ニュース/論文/ブログ（重要）」vs「Amazon/イベント/アーカイブ（ノイズ）」、重要リンクは何本あるか
  - `追加分析項目`: もう1記事（2〜3日前のもの）も同様にリンク分析して、重複しているドメインや記事があるか確認
  - `ページ構造情報`: 該当なし
  - `出力形式`: 具体的なURL込みで報告、200語以内

**実際のプロンプト**
```
指定のサイトを詳しく調べてください。

1. RSSフィード {サイトURL}/feed/ をフェッチして、最新5件の記事タイトル・URL・日付を列挙
2. 最新記事を1本フルでフェッチして以下を分析:
   - 本文の構造（見出し、段落数）
   - 外部リンクを全部列挙（アンカーテキスト + URL）
   - リンクをカテゴリ分類: 「ニュース/論文/ブログ（重要）」vs「Amazon/イベント/アーカイブ（ノイズ）」
   - 重要リンクは何本あるか

3. もう1記事（2〜3日前のもの）も同様にリンク分析して、重複しているドメインや記事があるか確認

結果を具体的なURL込みで報告してください。200語以内。
```
