# 🔷 スキル定義: ソース検証と整合性確認

**説明**: 収集したソースの引用・解釈の正しさをダブルチェックするスキル
**ステータス**: ✅ 公開中 | **ソース数**: 6 件 | **カテゴリ**: ソース検証と整合性確認

---

## テンプレートプロンプト（抽象）

```
以下の{検証対象}について、{参照ソース}の内容と整合性を確認してください。具体的には、{確認ポイント}に注目し、{期待する結果}を達成するために、{追加条件}を考慮した上で検証を行ってください。
```

## 変数一覧

- `{{検証対象}}`: 検証したいデータや成果物（例: 検証結果、特定のフォルダの内容）
- `{{参照ソース}}`: 検証の基準となるソース（例: 調査結果の見出し、原著、指定されたリサーチ結果）
- `{{確認ポイント}}`: 具体的にチェックすべき項目（例: 引用の正確さ、解釈の誤り、管理の有無）
- `{{期待する結果}}`: 検証後に達成したい状態（例: 誤りがないことの確認、ダブルチェック完了）
- `{{追加条件}}`: 検証時に考慮すべき制約や前提（例: 循環確認ではなくダブルチェック、対応関係の検証）

## 活用シーン

- レポート作成時の引用元と本文の整合性確認
- 調査結果と実際のソースページの突き合わせ検証
- 生成コンテンツの事実誤認チェック

# 🔶 具体インスタンス

### インスタンス 1: factcheckダブルチェック

> 元プロンプト: *現状の検証は、選んだソースを正しく解釈しているのかのチェックです　本当の事実確認はそれ以上追求が難しいので別の作業にするとして、今は集めたソースの参照や引用に間違いがないことの確認ができれば良い　つまり循環ではなく、あくまでダブルチェックで*

**変数の値**
  - `検証対象`: 検証結果
  - `参照ソース`: 選んだソース
  - `確認ポイント`: ソースの解釈が正しいかどうか
  - `期待する結果`: 集めたソースの参照や引用に間違いがないことの確認
  - `追加条件`: 循環ではなくあくまでダブルチェック、原著で検証も同じでソースに対して嘘をついていないかの確認

**実際のプロンプト**
```
以下の検証結果について、選んだソースの参照や引用に間違いがないことの確認をしてください。具体的には、ソースの解釈が正しいかどうかに注目し、本当の事実確認は別の作業として、今は集めたソースの参照や引用に間違いがないことの確認ができれば良いという期待する結果を達成するために、循環ではなくあくまでダブルチェックであり、原著で検証も同じでソースに対して嘘をついていないかの確認であるという追加条件を考慮した上で検証を行ってください。
```

### インスタンス 2: Web調査見出し検証

> 元プロンプト: *うーん　つまり　調査結果の見出しをあなたが提供したからうまくいったが、結局検索結果からこの見出しを見つけてきたわけではない？*

**変数の値**
  - `検証対象`: 調査結果の見出し
  - `参照ソース`: 検索結果
  - `確認ポイント`: 提供された見出しが実際に検索結果から取得されたものかどうか
  - `期待する結果`: 正しいソース経路を特定する
  - `追加条件`: 提供されたデータと検索結果の対応関係を検証する

**実際のプロンプト**
```
以下の調査結果の見出しについて、検索結果からこの見出しを見つけてきたわけではないという点を確認してください。具体的には、提供された見出しが実際に検索結果から取得されたものかどうかに注目し、正しいソース経路を特定するという期待する結果を達成するために、提供されたデータと検索結果の対応関係を検証するという追加条件を考慮した上で検証を行ってください。
```

### インスタンス 3: research_data管理確認

> 元プロンプト: *データフォルダに入っているものは、バージョン管理しないよね？*

**変数の値**
  - `検証対象`: データフォルダに入っているもの
  - `参照ソース`: バージョン管理のルール
  - `確認ポイント`: バージョン管理の対象外とするファイルの種類
  - `期待する結果`: 適切なバージョン管理設定を行う
  - `追加条件`: データの性質とチームの運用ルールを考慮する

**実際のプロンプト**
```
以下のデータフォルダに入っているものについて、バージョン管理しないという前提を確認してください。具体的には、バージョン管理の対象外とするファイルの種類に注目し、適切なバージョン管理設定を行うという期待する結果を達成するために、データの性質とチームの運用ルールを考慮するという追加条件を考慮した上で検証を行ってください。
```
