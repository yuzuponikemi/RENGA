# 🔷 スキル定義: ドキュメント作成・更新テンプレート

**説明**: ドキュメントの作成や更新を依頼する汎用プロンプト
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **カテゴリ**: ドキュメント作成・更新

---

## テンプレートプロンプト（抽象）

```
{対象ドキュメントの種類や目的}について、{具体的な要件や条件}を満たすように、{作業内容（作成・更新・修正など）}してください。また、{追加の確認事項や注意点}を考慮し、{成果物の形式や保存場所}に反映してください。
```

## 変数一覧

- `{{対象ドキュメントの種類や目的}}`: 作成・更新したいドキュメントの種類や目的（例: 使い方マニュアル、価格表記事、サイト設定ファイル）
- `{{具体的な要件や条件}}`: ドキュメントに含めるべき具体的な情報や条件（例: プロセスフロー、URLリンク、確認手順）
- `{{作業内容}}`: 行う作業の種類（例: 作成、更新、修正、追加）
- `{{追加の確認事項や注意点}}`: 作業時に注意すべき点や追加で確認すべきこと（例: サイトが壊れていないか確認、mdファイルを参照）
- `{{成果物の形式や保存場所}}`: 成果物の形式や保存先（例: mdファイル、設定ファイル、特定のフォルダ）

## 活用シーン

- 製品の使い方ドキュメントを作成する
- 価格表記事を最新情報に更新する
- サイト設定ファイルに記事を追加し、動作確認手順を記録する

### インスタンス 3: 記事追加とサイト確認手順記録

> 元プロンプト: *それで、前回追加した記事について{設定ファイル名}に追加されていないので、webサイトが壊れています　これは追加してください　また同じようにwebサイトが壊れていないか確認する方法を考え、 それを含む記事の追加方法をmdに記録しておいてくだ*

**変数の値**
  - `対象ドキュメントの種類や目的`: {設定ファイル名}設定ファイルと記事追加手順のmdファイル
  - `具体的な要件や条件`: webサイトが壊れていないか確認する方法を含める
  - `作業内容`: 修正（{設定ファイル名}に追加）および記録（mdファイル作成）
  - `追加の確認事項や注意点`: 記事追加依頼時は必ずmdを読むように工夫する
  - `成果物の形式や保存場所`: {設定ファイル名}（設定ファイル）とmdファイル

**実際のプロンプト**
```
それで、前回追加した記事について{設定ファイル名}に追加されていないので、webサイトが壊れています。これは追加してください。また同じようにwebサイトが壊れていないか確認する方法を考え、それを含む記事の追加方法をmdに記録しておいてください。そして記事の追加を依頼されたときは必ずそのmdを読むように工夫してください。
```
