# パイプライン分析・構築

**説明**: データパイプラインの課題分析、設計、実行指示を行うスキル
**ステータス**: ✅ 公開中 | **ソース数**: 8 件 | **ユニークユーザー**: 7 人 | **カテゴリ**: パイプライン分析・構築

---

## テンプレートプロンプト

```
{パイプライン対象} について、{分析内容} を実行し、{期待する結果} を確認したい。{追加条件} があれば考慮して、{次のアクション} を進めてください。
```

## 変数一覧

- `{{パイプライン対象}}`: 分析・構築するパイプラインの名称やURL（例: 動画URL、リポジトリ名、処理フロー名）
- `{{分析内容}}`: 実行する具体的な分析や処理内容（例: 課題の発見、現状のまとめ、パイプライン実行）
- `{{期待する結果}}`: 得たいアウトプット（例: 課題リスト、実行結果の確認、レポート）
- `{{追加条件}}`: リソース制約や優先順位などの補足条件（例: リソースが空いたら実行、未処理の停止）
- `{{次のアクション}}`: 分析後の次のステップ（例: 統合、設計、パブリッシュ）

## 活用シーン

- 既存のデータパイプラインのボトルネックを特定したいとき
- 新しいパイプラインを設計・構築するとき
- 複数のパイプラインを統合・最適化するとき

## トリガーフレーズ

- `パイプラインに統合して`

## 具体インスタンス

### インスタンス 1: 動画パイプライン課題分析

> 元プロンプト: *{指定した動画URL} これをプロセスしてみて、このパイプラインの課題を見つけてみて*

**変数の値**
  - `パイプライン対象`: {指定した動画URL}
  - `分析内容`: このパイプラインの課題を見つける
  - `期待する結果`: 課題リスト
  - `追加条件`: 特に制約はない
  - `次のアクション`: 改善提案

**実際のプロンプト**
```
{指定した動画URL} について、このパイプラインの課題を見つける を実行し、課題リスト を確認したい。特に制約はない ので、次のアクションとして改善提案 を進めてください。
```
