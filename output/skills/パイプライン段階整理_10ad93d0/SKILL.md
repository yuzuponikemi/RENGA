## テンプレートプロンプト

```
{作業対象}について、まず{現状確認項目}を振り返りたい。次に{優先事項}を進めるため、{次のアクション}を検討してほしい。また、{並行作業}として{追加タスク}も設計したい。
```

## 変数一覧

- `{{作業対象}}`: パイプラインの対象（例: 新着コンテンツの自動処理）
- `{{現状確認項目}}`: 確認すべきポイント（例: 現在の機能状況、テスト進捗）
- `{{優先事項}}`: 優先して取り組む内容（例: 既存の処理済みコンテンツのカバー）
- `{{次のアクション}}`: 具体的な次のステップ（例: 統合テストの実施）
- `{{並行作業}}`: 同時に進める作業の概要（例: バックグラウンドで進めたい処理）
- `{{追加タスク}}`: 並行作業の詳細（例: 新着コンテンツの自動処理パイプライン設計と定期実行）

## 活用シーン

- データ処理パイプラインの現状整理と優先順位付け
- CI/CDパイプラインの段階ごとの進捗確認と次工程設計
- コンテンツ生成パイプラインの自動化設計

## 具体インスタンス

### インスタンス 1: Podcastパイプライン整理

> 元プロンプト: *ここまでをまず振り帰りたいです　今何ができるのか、そのパイプラインとしてはどこまでテストしたのか？統合テストでまず確認すべきではないか*

**変数の値**
  - `作業対象`: 新着コンテンツの自動記事化処理
  - `現状確認項目`: 今何ができるのか、そのパイプラインとしてはどこまでテストしたのか？
  - `優先事項`: すでに公開されている処理済みのコンテンツたちについて、こちらのパイプラインでカバーする
  - `次のアクション`: 統合テストでまず確認すべきではないか
  - `並行作業`: 裏でやっておきたい処理
  - `追加タスク`: 新着コンテンツの自動記事化処理のパイプライン設計と定期実行

**実際のプロンプト**
```
新着コンテンツの自動記事化処理について、まず今何ができるのか、そのパイプラインとしてはどこまでテストしたのか？を振り返りたい。次にすでに公開されている処理済みのコンテンツたちについて、こちらのパイプラインでカバーしたい。そのため、統合テストでまず確認すべきではないか。また、裏でやっておきたいのが、新着コンテンツの自動記事化処理です。パイプラインしっかり作って定期実行させたいので、設計してみて。
```

### インスタンス 2: 音声文字起こしパイプライン

> 元プロンプト: *過去にダウンロードした音声データそのものです　これを音声認識ツールなどで文字起こししてから　これまでと同様のパイプラインに渡すつもりで開発途中までやっていました*

**変数の値**
  - `作業対象`: 過去にダウンロードした音声データ
  - `現状確認項目`: 現状のデータ状態
  - `優先事項`: 文字起こし処理を優先して進める
  - `次のアクション`: 音声認識ツールなどで文字起こししてからこれまでと同様のパイプラインに渡す設計を検討
  - `並行作業`: 新着コンテンツの自動取得処理
  - `追加タスク`: 新着コンテンツの自動取得パイプライン設計

**実際のプロンプト**
```
過去にダウンロードした音声データについて、まず現状のデータ状態を振り返りたい。次に文字起こし処理を優先して進めるため、音声認識ツールなどで文字起こししてからこれまでと同様のパイプラインに渡す設計を検討してほしい。また、並行して新着コンテンツの自動取得処理も設計したい。
```
