# 🔷 スキル定義: ファイル構造解析テンプレート

**説明**: ファイルの構造（見出し・セクション）を解析し、チャンク分割計画を生成する
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **カテゴリ**: ファイル構造解析

---

## テンプレートプロンプト（抽象）

```
以下のファイルを解析し、その構造を詳細に把握してください。

対象ファイル: {file_path}

必要な情報:
1. すべてのトップレベル見出し（{heading_marker}）とその行番号
2. セクション区切りマーカー（{section_dividers}）の有無と位置
3. 各見出し間の概算文字数
4. 各見出し直後の最初の{lines_after_heading}行の内容
5. ファイル全体の総文字数

目的: この{file_size}のファイルを、LLM分析パイプラインで個別処理するための適切なチャンク数（理想的には{target_chunks}）に分割する方法を特定すること。

出力形式: 見出し → 行番号 → 概算長さ → 簡単な説明 の完全なマッピングを返してください。
```

## 変数一覧

- `{{file_path}}`: 解析対象ファイルの絶対パスまたは相対パス（例: /Users/あなたのユーザ名/source/project/data/example.md）
- `{{heading_marker}}`: 見出しを示すマーカー記号（例: #、##、===）
- `{{section_dividers}}`: セクション区切りとして検索する文字列のリスト（例: Part, Section, ---）
- `{{lines_after_heading}}`: 各見出し直後に取得する行数（例: 10）
- `{{file_size}}`: ファイルのサイズ説明（例: 1MB、500KB）
- `{{target_chunks}}`: 目標とする分割チャンク数（例: 6-8）

## 活用シーン

- 長文のファイルをLLM処理用に適切なチャンクに分割する計画を立てる
- Webページやドキュメントのセクション構造を解析し、テストアサーションを作成する
- Markdownドキュメントの目次を自動生成するための構造把握

### インスタンス 2: Wikipedia章構成調査

> 元プロンプト: *I'm building a book chapter resolver that fetches structure from Wikipedia's sections API. I need to know what chapter/s*

**変数の値**
  - `file_path`: Wikipedia API（https://en.wikipedia.org/w/api.php?action=parse&page=指定した記事タイトル&prop=sections&format=json）
  - `heading_marker`: toclevel 1-2
  - `section_dividers`: References, See also, External links
  - `lines_after_heading`: 5
  - `file_size`: 5記事
  - `target_chunks`: 実際の章数

**実際のプロンプト**
```
以下のファイルを解析し、その構造を詳細に把握してください。

対象ファイル: {file_path}

必要な情報:
1. すべてのトップレベル見出し（toclevel 1-2）とその行番号（lineフィールド）
2. セクション区切りマーカー（References, See also, External links）の有無と位置
3. 各見出し間の概算文字数
4. 各見出し直後の最初の5行の内容
5. ファイル全体の総文字数

目的: この5記事のファイルを、LLM分析パイプラインで個別処理するための適切なチャンク数（理想的には実際の章数）に分割する方法を特定すること。

出力形式: 見出し → 行番号 → 概算長さ → 簡単な説明 の完全なマッピングを返してください。
```
