# 🔷 スキル定義: フォルダ構造調査

**説明**: プロジェクトのフォルダ構成とファイル内容を網羅的に調査する
**ステータス**: ✅ 公開中 | **ソース数**: 6 件 | **カテゴリ**: フォルダ構造調査

---

## テンプレートプロンプト（抽象）

```
Explore the codebase at {target_path} thoroughly. I need to understand:
1. All files and their structure
2. The contents of {key_files} (full content)
3. How {key_component} currently works end-to-end
4. The full contents of the {specific_folder} folder (all subdirectories and files)
5. Any config files at the top level (like {config_file_patterns})

Please show me the complete file tree for {specific_folder} and return the full content of each file, organized by directory. I want to understand the architectural patterns and conventions used.
```

## 変数一覧

- `{{target_path}}`: 調査対象のプロジェクトパス（例: /Users/{ユーザー名}/source/{プロジェクト名}）
- `{{key_files}}`: 特に内容を確認したいファイルのカンマ区切りリスト（例: main.py, config.py, cli.py）
- `{{key_component}}`: 理解したい主要コンポーネントや機能の説明（例: パイプライン処理、CLIの動作）
- `{{specific_folder}}`: 詳細に調査したい特定フォルダ名（例: src/, tests/, {プロジェクト名}/）
- `{{config_file_patterns}}`: 設定ファイルのパターン（例: main.py, pyproject.toml, requirements.txt）

## 活用シーン

- 新しいメンバーがプロジェクトに参加した際のコードベース理解
- リファクタリング前の現状把握とアーキテクチャ分析
- 特定機能の実装パターンを既存コードから学習するため
