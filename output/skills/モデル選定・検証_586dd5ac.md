# 🔷 スキル定義: モデル選定・検証

**説明**: 複数のLLMモデルを比較・検証し、最適なモデルを選定するスキル
**ステータス**: 🔒 審査中 | **ソース数**: 5 件 | **カテゴリ**: モデル選定・検証

---

## テンプレートプロンプト（抽象）

```
{モデル候補}の中で、{重視する観点}を考慮して最適なモデルを選定・検証してください。{追加の条件}があればそれも加味してください。
```

## 変数一覧

- `{{モデル候補}}`: 比較・検証したいLLMモデル名のリスト（例: qwen3, llama3.2, gpt-4）
- `{{重視する観点}}`: モデル選定で重視する基準（例: 精度、速度、ハルシネーションの少なさ、コスト）
- `{{追加の条件}}`: その他の条件や注意点（例: 日本語対応、実行時間の制約、データ品質のチェック）

## 活用シーン

- 複数のLLMモデルを比較して、特定タスクに最適なモデルを選びたいとき
- 新しいモデルがリリースされたので、既存モデルと性能比較したいとき
- コストと精度のバランスを考慮して、本番環境に導入するモデルを決めたいとき

# 🔶 具体インスタンス

### インスタンス 1: qwen3最高精度検証

> 元プロンプト: *うーん　時間かかってもいいから、{モデル候補}ばっかり使って、最高精度でやった時にどうなるかみてみましょう　方法序説でもう一回{モデル候補}で実行してみて　前回のRUNには、変な文字が混じっていたり、日本語と英語が混ざってたりしたので、そう*

**変数の値**
  - `モデル候補`: qwen3
  - `重視する観点`: 最高精度
  - `追加の条件`: 時間制限なし、前回のRUNで発生した文字化け・言語混在の問題をチェック

**実際のプロンプト**
```
{モデル候補}の中で、最高精度を重視して最適なモデルを選定・検証してください。時間がかかっても構いません。前回のRUNでは変な文字の混入や日本語と英語の混在があったので、そうした問題が起きないかもチェックしてください。
```
