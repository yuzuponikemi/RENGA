# 🔷 スキル定義: リポジトリ探索分析

**説明**: リポジトリ構造・ノートブック内容・規約を網羅的に調査するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 7 件 | **カテゴリ**: リポジトリ探索分析

---

## テンプレートプロンプト（抽象）

```
以下のリポジトリを徹底的に探索・分析してください。

## 対象リポジトリ
{repository_path}

## 調査項目
1. **ディレクトリ構造**: ルート直下の全フォルダ・ファイル構成。{target_subdirectory} のサブディレクトリ一覧と各ディレクトリ内のファイル数。
2. **ノートブック分析**: 以下の観点で複数の代表的なノートブックを調査
   - ファイル名と番号付けパターン
   - 最初のマークダウンセルのタイトル（{title_format}）
   - 難易度（{difficulty_format}）
   - 推定学習時間
   - 学習目標（チェックリスト項目）
   - 主要トピック・キーワード
   - 使用言語（日本語/英語）
   - コードスタイル・インポートパターン
3. **特定トピックの既存コンテンツ**: {target_topic} に関連する既存ノートブックを全ディレクトリから検索し、内容の重複をチェック
4. **ドキュメントファイル**: 以下のファイルを読み込み、内容を要約
   - {doc_files}
5. **設定ファイル**: CLAUDE.md やその他の設定ファイルの有無と内容

## 出力形式
- 各調査項目ごとにセクションを分けて詳細に報告
- 重複やギャップがある場合は明記
- ノートブックのテンプレート形式（ヘッダー構造・セクション構成）があれば正確に記述
```

## 変数一覧

- `{{repository_path}}`: 調査対象のリポジトリの絶対パス
- `{{target_subdirectory}}`: 特に詳しく見たいサブディレクトリ
- `{{title_format}}`: ノートブックのタイトル形式の説明
- `{{difficulty_format}}`: 難易度の表記形式
- `{{target_topic}}`: 調査したい特定トピック
- `{{doc_files}}`: 読み込むべきドキュメントファイルのリスト（カンマ区切り）

## 活用シーン

- 新しいノートブックセクションを追加する前に、既存コンテンツとの重複をチェックする
- リポジトリ全体の構成を把握し、カリキュラムの一貫性を確認する
- 特定トピックに関する既存の学習リソースを棚卸しする
