# リポジトリ構造整理確認

**説明**: 複数リポジトリの構成・状態を整理し確認するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 6 件 | **ユニークユーザー**: 3 人 | **カテゴリ**: リポジトリ構造の整理確認

---

## テンプレートプロンプト

```
以下のリポジトリ構造について、現在の状況を整理し、問題点を確認してください。

- ルートリポジトリ: {root_repo_name}
- 内部に含まれるサブリポジトリ: {sub_repo_names}
- マウント方法: {mount_method}
- 懸念事項: {concerns}

具体的に確認してほしい点:
1. 各リポジトリの現在のブランチ状態
2. リモートとの同期状態（未プッシュのコミット有無）
3. ブランチが切られたまま放置されていないか
4. サブリポジトリの配置場所とルートリポジトリとの関係
5. ツールや機能の追加が正しく反映されているか
6. この構成での正しい使い方・運用フロー
```

## 変数一覧

- `{{root_repo_name}}`: ルートとなるリポジトリ名
- `{{sub_repo_names}}`: 内部に含まれるサブリポジトリ名のリスト（入れ子構造の場合はその旨を含む）
- `{{mount_method}}`: リポジトリのマウント方法や運用方法（例: Dockerマウント、シンボリックリンク、エージェント経由の追加依頼など）
- `{{concerns}}`: ユーザーが懸念している具体的な問題点（例: ブランチ放置、リモート同期、使い方の理解不足、配置場所の不明確さ）

## 活用シーン

- 複数のGitリポジトリが入れ子になっているプロジェクトの整理・状態確認
- Dockerやエージェント経由でマウントしたリポジトリ群の同期状態やブランチ管理の確認
- 新しく追加したツールや機能が正しく配置・反映されているかの確認

## トリガーフレーズ
