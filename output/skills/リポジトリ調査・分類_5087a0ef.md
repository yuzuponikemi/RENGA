# 🔷 スキル定義: リポジトリ調査・分類スキル

**説明**: 複数リポジトリを調査し、概要・技術スタック・状態を分類する
**ステータス**: ✅ 公開中 | **ソース数**: 7 件 | **カテゴリ**: リポジトリ調査・分類

---

## テンプレートプロンプト（抽象）

```
以下のリポジトリ群を調査し、各リポジトリについて以下の情報を提供してください。

調査対象ディレクトリ: {target_directory}
リポジトリ一覧: {repo_list}

各リポジトリについて、以下のファイルを読み込んで調査してください:
1. README.md（または readme.md, README）
2. CLAUDE.md（存在する場合）
3. READMEがない場合は package.json / pyproject.toml / go.mod の name/description フィールド

各リポジトリに対して以下を出力:
- 名前
- 一行説明（何をするものか）
- 技術スタック（主要言語/フレームワーク）
- 状態（active/experiment/archived/template）
- カテゴリ（例: AI/ML tool, game, web app, agent framework, MCP server, data pipeline など）

実際のファイル内容を読み、推測ではなく事実に基づいて報告してください。
```

## 変数一覧

- `{{target_directory}}`: 調査対象のルートディレクトリの絶対パス（例: /Users/{ユーザー名}/source/{プロジェクト名}）
- `{{repo_list}}`: 調査するリポジトリ名のカンマ区切りリスト（例: repo1, repo2, repo3）

## 活用シーン

- 個人のGitHubリポジトリフォルダを一括整理・分類したいとき
- チームの複数プロジェクトの現状把握・棚卸しをするとき
- 新しく参加したプロジェクト群の全体像を素早く把握したいとき
