## テンプレートプロンプト

```
{プロジェクトパス}を{操作内容}して、{追加要件}。さらに{確認事項}を{実行方法}で行ってください。
```

## 変数一覧

- `{{プロジェクトパス}}`: 操作対象のローカルディレクトリパス（例: /Users/あなたのユーザ名/source/プロジェクト名）
- `{{操作内容}}`: 実行する具体的な操作（例: ローカルドキュメントとして処理、開発、git管理）
- `{{追加要件}}`: 操作に付随する追加条件（例: ドキュメントを参照しないと回答できないクエリを追加）
- `{{確認事項}}`: 検証・確認したい内容（例: 検索能力、イシュー一覧、PR作成）
- `{{実行方法}}`: 操作を実行する手段やツール（例: Ollamaテスト、テレグラム経由、nanoclaw経由）

## 活用シーン

- ローカルプロジェクトのドキュメントを読み込み、AIに検索能力をテストさせる
- Git管理されたプロジェクトをテレグラム経由で操作し、イシューからPRを作成する
- ローカルディレクトリの読み書き権限を設定し、サンドボックス環境で安全に開発する

### インスタンス 2: Gitプロジェクトのテレグラム開発

> 元プロンプト: *たとえば　/Users/{ユーザ名}/source/personal/{プロジェクト名}　は私が開発中のプロジェクトでgit管理しています　これをnanoclaw経由で開発してみたい　テレグラム経由で　{プロジェクト名}のイシュー一覧を表示*

**変数の値**
  - `プロジェクトパス`: /Users/{ユーザ名}/source/personal/{プロジェクト名}
  - `操作内容`: git管理のプロジェクトをnanoclaw経由で開発
  - `追加要件`: テレグラム経由でイシュー一覧を表示、ピックアップして実装、PR作成
  - `確認事項`: イシュー一覧、実装、PR
  - `実行方法`: nanoclaw経由、テレグラム経由

**実際のプロンプト**
```
{プロジェクトパス}は私が開発中のプロジェクトでgit管理しています。これを{実行方法}経由で開発してみたい。テレグラム経由で{プロジェクト名}のイシュー一覧を表示して、そこから一つピックアップして実装してPR出しておいて、という使い方をしてください。
```
