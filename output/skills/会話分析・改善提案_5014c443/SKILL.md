# 会話分析・改善提案

**説明**: 会話ログやプロンプトを分析し、改善案を提案するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 会話分析・改善提案

---

## テンプレートプロンプト

```
あなたは{役割}です。以下の{対象}を分析し、{分析観点}に基づいて{改善目的}を提案してください。特に{注目ポイント}に注意し、{制約条件}を考慮した上で、{出力形式}で回答してください。
```

## 変数一覧

- `{{役割}}`: 分析者の立場や専門性（例: 会話分析の専門家、AI利用ログの解析者）
- `{{対象}}`: 分析するデータやコンテンツ（例: ポッドキャストの会話ログ、AIとの会話履歴）
- `{{分析観点}}`: 分析の視点や基準（例: 会話の連続性、事実確認の正確性、引用の妥当性）
- `{{改善目的}}`: 改善のゴール（例: 会話の質向上、ワークフローの効率化、引用ミスの防止）
- `{{注目ポイント}}`: 特に注目すべき要素（例: 特定の人物の登場理由、ソースの解釈誤り、文字化けの原因）
- `{{制約条件}}`: 考慮すべき制約（例: 原著での検証、Webバージョンでの実装可能性、他のプロジェクトへの応用）
- `{{出力形式}}`: 回答の形式（例: 改善プラン、パイプライン図、具体的な手順リスト）

## 活用シーン

- ポッドキャストの会話ログから役割分担を分析し、改善案を提案
- AIとの会話履歴から連続性をトラックする方法を調査
- 論文執筆における引用の正確性をチェックするパイプラインを設計

## 具体インスタンス

### インスタンス 1: ポッドキャスト分析

> 元プロンプト: *ありがとう　concept-graphはかなりいい感じですね！　しかしscriptsをみると、なぜかデカルトが登場しています・・・　これはなぜでしょうか　ポッドキャストの会話ですが、毎回教授と学生みたいに分けるでいいと思います*

**変数の値**
  - `役割`: 会話分析の専門家
  - `対象`: ポッドキャストの会話ログ
  - `分析観点`: 会話の役割分担
  - `改善目的`: 改善案の提案
  - `注目ポイント`: 特定の人物が登場している理由
  - `制約条件`: 毎回{役割A}と{役割B}のように分ける形式
  - `出力形式`: 具体的な改善プラン

**実際のプロンプト**
```
あなたは会話分析の専門家です。以下のポッドキャストの会話ログを分析し、会話の役割分担に基づいて改善案を提案してください。特に特定の人物が登場している理由に注意し、毎回{役割A}と{役割B}のように分ける形式を考慮した上で、具体的な改善プランで回答してください。
```

### インスタンス 2: Gemini会話連続性調査

> 元プロンプト: *了解です　私と同じように、geminiの会話ログを全て集めて分析しようとしている人がどこかにいるはずです　何か会話の連続性を正確にトラックする方法がないものか、調べてみてもらえますか　gemini webサイトを操作することで　このエントリ*

**変数の値**
  - `役割`: AI利用ログの解析者
  - `対象`: AIとの会話履歴
  - `分析観点`: 会話の連続性を正確にトラックする方法
  - `改善目的`: 調査結果の提案
  - `注目ポイント`: エントリ同士の連続性を確認する方法
  - `制約条件`: {サービス名}のWebサイトを操作する方法も視野に入れる
  - `出力形式`: 具体的な手順リスト

**実際のプロンプト**
```
あなたはAI利用ログの解析者です。以下のAIとの会話履歴を分析し、会話の連続性を正確にトラックする方法に基づいて調査結果を提案してください。特にエントリ同士の連続性を確認する方法に注意し、{サービス名}のWebサイトを操作する方法も視野に入れた上で、具体的な手順リストで回答してください。
```

### インスタンス 3: 引用チェックパイプライン

> 元プロンプト: *現状のfactcheckは、選んだソースを正しく解釈しているのかのチェックです　本当の事実確認はそれ以上追求が難しいので別プロジェクトにするとして、今は集めたソースの参照や引用に間違いがないことの確認ができれば良い　つまり循環ではなく、あく*

**変数の値**
  - `役割`: 学術引用の専門家
  - `対象`: 事実確認プロセス
  - `分析観点`: ソースの参照や引用の正確性
  - `改善目的`: パイプラインの設計
  - `注目ポイント`: 原著で検証すること
  - `制約条件`: Webバージョンでも実装可能で論文執筆の引用にも応用できること
  - `出力形式`: パイプライン図

**実際のプロンプト**
```
あなたは学術引用の専門家です。以下の事実確認プロセスを分析し、ソースの参照や引用の正確性に基づいてパイプラインを設計してください。特に原著で検証することに注意し、Webバージョンでも実装可能で論文執筆の引用にも応用できることを考慮した上で、パイプライン図で回答してください。
```
