# 🔷 スキル定義: 作業方針の調整と指示

**説明**: ユーザーが作業方針や設定を変更し、再実行を指示するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **カテゴリ**: 作業方針の調整と指示

---

## テンプレートプロンプト（抽象）

```
{状況認識}に基づき、{変更内容}を実施してください。{前提条件}を確認した上で、{追加指示}を行い、{再実行依頼}をお願いします。
```

## 変数一覧

- `{{状況認識}}`: 現在の状況や問題点の説明（例: 選んだ本が微妙だった、エラーが出ている）
- `{{変更内容}}`: 具体的に変更したい方針や設定（例: 本を切り替える、英語で回答する）
- `{{前提条件}}`: 変更前に確認すべき条件や環境設定（例: APIキーの有無、ファイルの存在）
- `{{追加指示}}`: 変更に伴う追加の作業指示（例: 不要ファイルの削除、git管理の調整）
- `{{再実行依頼}}`: 変更後に何を再実行してほしいかの依頼（例: 再テスト、ページ作成、確認）

## 活用シーン

- 作業中にエラーが発生した場合、方針を変更して再実行を依頼する
- 使用するツールやAPIを変更し、その設定確認を依頼する
- 出力形式や言語を変更し、再生成を依頼する
