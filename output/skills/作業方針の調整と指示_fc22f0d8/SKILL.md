# 作業方針の調整と指示

**説明**: ユーザーが作業方針や設定を変更し、具体的な指示を出すスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 作業方針の調整と指示

---

## テンプレートプロンプト

```
{状況認識}。{変更内容}。{追加条件}。{確認依頼}。
```

## 変数一覧

- `{{状況認識}}`: 現在の状況や問題点の説明（例: 選んだ本が期待と合わなかった）
- `{{変更内容}}`: 具体的に変更したい方針や設定（例: 別の本に切り替える）
- `{{追加条件}}`: 変更に伴う追加の条件や注意点（例: 指定したAPIキーが利用可能）
- `{{確認依頼}}`: 実行前に確認してほしいこと（例: まずAPIが使えるか確認してから再テスト）

## 活用シーン

- プロジェクトの技術スタックを変更したいとき
- ファイル管理方法を手動から自動に切り替えたいとき
- Git管理対象のファイルを調整したいとき

## トリガーフレーズ

- `方針を変更して`
- `設定を変えて`
- `やり方を変えたい`
- `確認してから進めて`
- `条件を追加して`

### インスタンス 1: 本の変更と再テスト

> 元プロンプト: *うーん、選んだ本が微妙でした、すみません　千葉雅也さんの　勉強の哲学　という本に切り替えて試してみましょう　一応私はTavily api keyを.envファイルに記載しているので使えるはずです　まずtavilyが使えるかどうか確認してから*

**変数の値**
  - `状況認識`: 選んだ本が期待と合いませんでした
  - `変更内容`: 別の本に切り替えて試してみましょう
  - `追加条件`: 一応あなたは指定したAPIキーを環境変数ファイルに記載しているので使えるはずです
  - `確認依頼`: まずAPIが使えるかどうか確認してから、再テストお願いします

**実際のプロンプト**
```
選んだ本が期待と合いませんでした。別の本に切り替えて試してみましょう。一応あなたは指定したAPIキーを環境変数ファイルに記載しているので使えるはずです。まずAPIが使えるかどうか確認してから、再テストお願いします。
```
