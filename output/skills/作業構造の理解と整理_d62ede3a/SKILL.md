# 作業構造の理解と整理

**説明**: プロジェクト構成やワークフローを整理し、最適な作業構造を設計する
**ステータス**: ✅ 公開中 | **ソース数**: 8 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 作業構造の理解と整理

## テンプレートプロンプト

```
{作業対象}の{構造要素}をよくわかっていないんやけど、そこには{設定ファイル}は置けないですよね。今全部の{リポジトリ群}が置いてある例えば{統合用フォルダ}をひとつ新たに統括用に作っておいて、そいつの中に{フォルダ}を移せばいいのかな。今いらない{不要要素}も残っちゃっているので。{追加質問}
```

## 変数一覧

- `{{作業対象}}`: 整理したいシステムやツール名（例: ワークスペース機能）
- `{{構造要素}}`: 理解したい構造の部分（例: 機能の仕組み）
- `{{設定ファイル}}`: 配置したい設定ファイル名（例: 設定ファイル）
- `{{リポジトリ群}}`: 現在のリポジトリの配置場所（例: 全てのリポジトリ）
- `{{統合用フォルダ}}`: 新しく作る統括フォルダ名（例: 統括用フォルダ）
- `{{フォルダ}}`: 移動対象のフォルダ（例: プロジェクトフォルダ）
- `{{不要要素}}`: 整理したい不要なもの（例: 不要なプロジェクト）
- `{{追加質問}}`: 追加で確認したいこと（例: このやり方であっているか？）

## 活用シーン

- 新しいプロジェクトのディレクトリ構成を設計するとき
- 既存のワークスペース設定を整理・最適化するとき
- 複数プロジェクトの管理方法を統一したいとき
