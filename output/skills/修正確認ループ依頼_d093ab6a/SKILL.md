# 修正確認ループ依頼

**説明**: 修正指示と確認依頼を繰り返し実行するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 7 件 | **ユニークユーザー**: 4 人 | **カテゴリ**: 修正・確認依頼

---

## テンプレートプロンプト

```
{指示内容} を実行して、{確認対象} を確認してください。{追加条件} がある場合はそれも反映してください。必要に応じて {成功条件} になるまで修正と確認を繰り返してください。
```

## 変数一覧

- `{{指示内容}}`: 実行してほしい具体的な作業内容（例: 最初の記事を作成）
- `{{確認対象}}`: 確認してほしい項目（例: 色が直っているか、返事が来ているか）
- `{{追加条件}}`: 追加で反映してほしい条件（例: タイトルに特定の言葉を含める）
- `{{成功条件}}`: 繰り返しの終了条件（例: 返事が返ってくる、改善が確認できる）

## 活用シーン

- コードの修正と動作確認を繰り返してほしいとき
- デザインの修正と確認をループしてほしいとき
- 外部APIの応答確認と再試行を自動化したいとき

## 具体インスタンス

### インスタンス 1: Substack記事作成と色修正

> 元プロンプト: *Pushしました　最初の記事を作って見てください　まずは　ブログの紹介で*

**変数の値**
  - `指示内容`: 最初の記事を作成（ブログの紹介）
  - `確認対象`: 色が直っているか
  - `追加条件`: タイトルなどに書くときは指定したフレーズという言葉を含める
  - `成功条件`: 色が正しく、タイトルに指定したフレーズが含まれている

**実際のプロンプト**
```
最初の記事を作成してください。まずはブログの紹介で。色が直っているか確認してください。タイトルなどに書くときは特定の言葉だけじゃなくて、指定したフレーズという言葉に入れといてもらえるとわかりやすい。
```

---

### インスタンス 2: 返信確認ループ修正

> 元プロンプト: *やっぱり返信こないですね・・・　ログを見ながら、返事が帰ってくるか確認して、成功するまで修正し続けてください*

**変数の値**
  - `指示内容`: ログを見ながら返事が返ってくるか確認
  - `確認対象`: 返事が返ってきているか
  - `追加条件`: なし
  - `成功条件`: 返事が返ってくる

**実際のプロンプト**
```
ログを見ながら、返事が返ってくるか確認してください。返事が返ってこない場合は、成功するまで修正し続けてください。
```

---

### インスタンス 3: モデル設定とレポート確認

> 元プロンプト: *基本は指定したモデルにして、defaultでmaxのコンテキストを処理するようにしましょう　それで同じクエリに対するレポートを確認して、改善しているかチェックしてください*

**変数の値**
  - `指示内容`: 基本は指定したモデルにして、defaultでmaxのコンテキストを処理するように設定
  - `確認対象`: 同じクエリに対するレポート
  - `追加条件`: 改善しているかチェック
  - `成功条件`: 改善が確認できる

**実際のプロンプト**
```
基本は指定したモデルにして、defaultでmaxのコンテキストを処理するように設定してください。それで同じクエリに対するレポートを確認して、改善しているかチェックしてください。
```
