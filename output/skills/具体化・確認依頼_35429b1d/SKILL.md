# 具体化・確認依頼

**説明**: 抽象的な内容を具体化し、確認や詳細説明を求めるスキル
**ステータス**: ✅ 公開中 | **ソース数**: 9 件 | **ユニークユーザー**: 7 人 | **カテゴリ**: 具体化・確認依頼

---

## テンプレートプロンプト

```
以下の内容について、{対象} の {確認ポイント} を具体的に教えてください。

背景: {背景情報}

特に知りたいこと:
1. {質問1}
2. {質問2}
3. {質問3}

{追加条件} があればそれも考慮して回答してください。
```

## 変数一覧

- `{{対象}}`: 確認・具体化したい対象（例: スキル定義、ブログタイトル、システム構成）
- `{{確認ポイント}}`: 具体的に知りたい観点（例: 見た目、動作方法、決定基準）
- `{{背景情報}}`: 対象に関する前提や状況説明（例: ユーザーの属性、既存の状態）
- `{{質問1}}`: 最初の具体的な質問（例: どんな見た目か？）
- `{{質問2}}`: 2つ目の具体的な質問（例: どうやって決めるのか？）
- `{{質問3}}`: 3つ目の具体的な質問（例: 拡張方法は？）
- `{{追加条件}}`: 回答時に考慮すべき制約や希望（例: 無料で、英語対応可）

## 活用シーン

- 生成されたスキルの具体的な見た目や属性を確認したいとき
- ブログやメディアの方向性・タイトルを具体化したいとき
- システムの動作確認や修正手順を明確にしたいとき

## トリガーフレーズ

- `具体的にどんな感じ？`
- `どういうふうに決めるの？`
- `もっと詳しく教えて`

## 具体インスタンス

### インスタンス 1: スキル見た目確認

> 元プロンプト: *うーん　この結果作成されたスキルは、具体的にどんな見た目なんですか？中身を知りたい　けっこう抽象的にまとめられてる気がしていて　ただし具体が入るとより実用的にはシナジーうまれるので　スキルはこの両面で属性もって記録されるほうがいいのかもしれ*

**変数の値**
  - `対象`: スキル定義
  - `確認ポイント`: 具体的な見た目と中身
  - `背景情報`: この結果作成されたスキルは抽象的にまとめられている気がするが、具体が入ると実用的なシナジーが生まれる。スキルは両面で属性を持って記録されるべきかもしれない。
  - `質問1`: スキルは具体的にどんな見た目なのか？
  - `質問2`: 中身（属性や構造）はどうなっているのか？
  - `質問3`: 抽象と具体の両面をどう記録するのが良いか？
  - `追加条件`: 

**実際のプロンプト**
```
以下の内容について、スキル定義の具体的な見た目と中身を教えてください。

背景: この結果作成されたスキルは抽象的にまとめられている気がするが、具体が入ると実用的なシナジーが生まれる。スキルは両面で属性を持って記録されるべきかもしれない。

特に知りたいこと:
1. スキルは具体的にどんな見た目なのか？
2. 中身（属性や構造）はどうなっているのか？
3. 抽象と具体の両面をどう記録するのが良いか？

追加条件があればそれも考慮して回答してください。
```

### インスタンス 2: ブログ方向性決定

> 元プロンプト: *OK 今後このような　技術実験　英語最新情報のまとめ　Podcast深掘り　などなんでも書いていくメディアとして　どういうブログタイトルにするのが良いでしょうか　私という個人のペルソナと方向性をまず決めておいて、それをベースに今後の記事の書*

**変数の値**
  - `対象`: ブログタイトルと方向性
  - `確認ポイント`: 具体的な決め方
  - `背景情報`: 技術実験、英語最新情報のまとめ、Podcast深掘りなどなんでも書いていくメディアを考えている。現実の私は{専門分野}を大学で学び、{都市名}で{特定装置}のシステム開発を行うフルスタックエンジニア。実験系構築からソフトウェア開発、実験実行、結果解析まで全て回す。英語が得意で日本語も話せ、海外開発経験もある。
  - `質問1`: どういうブログタイトルが良いか？
  - `質問2`: ペルソナと方向性をどう決めるか？
  - `質問3`: 今後の記事の書き方や記憶の成長をどう設計するか？
  - `追加条件`: 人格を考えてまとめ、現実の私の経歴をベースにペルソナとしてやっていきたい。

**実際のプロンプト**
```
以下の内容について、ブログタイトルと方向性の具体的な決め方を教えてください。

背景: 技術実験、英語最新情報のまとめ、Podcast深掘りなどなんでも書いていくメディアを考えている。現実の私は{専門分野}を大学で学び、{都市名}で{特定装置}のシステム開発を行うフルスタックエンジニア。実験系構築からソフトウェア開発、実験実行、結果解析まで全て回す。英語が得意で日本語も話せ、海外開発経験もある。

特に知りたいこと:
1. どういうブログタイトルが良いか？
2. ペルソナと方向性をどう決めるか？
3. 今後の記事の書き方や記憶の成長をどう設計するか？

追加条件: 人格を考えてまとめ、現実の私の経歴をベースにペルソナとしてやっていきたい。
```

### インスタンス 3: ニュースレター設定確認

> 元プロンプト: *そもそもなんですけど　どの記事をどうニュースレター化するのかってどういうふうに決めるの？それはsubstack側の設定？あとニュースレター化するなら記事はsubstack側に書く必要があるの？　ニュースレターでメール送信して、私が今週更新し*

**変数の値**
  - `対象`: ニュースレター化の方法
  - `確認ポイント`: 具体的な決定基準と設定方法
  - `背景情報`: どの記事をどうニュースレター化するのか決め方、{サービス名}側の設定、記事の書き場所、メール送信内容について不明点がある。今週更新した記事一覧とリンクリストだけを流したい。
  - `質問1`: どの記事をニュースレター化するかどう決めるのか？それは{サービス名}側の設定か？
  - `質問2`: ニュースレター化するなら記事は{サービス名}側に書く必要があるのか？
  - `質問3`: メール送信で記事一覧とリンクリストだけを流すことは可能か？
  - `追加条件`: 無料で、一般公開で不特定多数が見る、英語コンテンツも考えている。

**実際のプロンプト**
```
以下の内容について、ニュースレター化の具体的な方法と決定基準を教えてください。

背景: どの記事をどうニュースレター化するのか決め方、{サービス名}側の設定、記事の書き場所、メール送信内容について不明点がある。今週更新した記事一覧とリンクリストだけを流したい。

特に知りたいこと:
1. どの記事をニュースレター化するかどう決めるのか？それは{サービス名}側の設定か？
2. ニュースレター化するなら記事は{サービス名}側に書く必要があるのか？
3. メール送信で記事一覧とリンクリストだけを流すことは可能か？

追加条件: 無料で、一般公開で不特定多数が見る、英語コンテンツも考えている。
```
