# 🔷 スキル定義: 作業再開・再実行

**説明**: 中断した作業の再開や、特定処理の再実行を指示する
**ステータス**: ✅ 公開中 | **ソース数**: 44 件 | **カテゴリ**: 再実行・再確認

---

## テンプレートプロンプト（抽象）

```
{作業対象}に対して{作業内容}を実行して。必要に応じて{追加条件}を考慮してください。
```

## 変数一覧

- `{{作業対象}}`: 処理対象となるファイル、プロジェクト、ドキュメントなど（例: ブログ記事、特定のドキュメント、README）
- `{{作業内容}}`: 実行する具体的なアクション（例: 再開、検証、更新、コピー追加）
- `{{追加条件}}`: 作業時の注意点や補足要件（例: 引用は数字で、本論はアブスト・本論・結論の構成に）

## 活用シーン

- 中断したコード生成やドキュメント作成を再開する
- CIに失敗したPRの修正を依頼する
- 複数ファイルにまたがる一括更新作業を指示する

---

# 🔶 具体インスタンス

### インスタンス 1: CI失敗修正依頼

> 元プロンプト: *githubのPRを見ると、CIに失敗していました、なぜか調べて修正して*

**変数の値**
  - `作業対象`: リポジトリのPR
  - `作業内容`: CI失敗の原因調査と修正
  - `追加条件`: なし

**実際のプロンプト**
```
リポジトリのPRを見ると、CIに失敗していました、なぜか調べて修正して
```

---

### インスタンス 2: ブログ一括追加

> 元プロンプト: *ブログにこれらの記事を追加するための作業を実行して*

**変数の値**
  - `作業対象`: ブログ
  - `作業内容`: 記事の追加
  - `追加条件`: これらの記事（コンテキスト内の記事群）

**実際のプロンプト**
```
ブログにこれらの記事を追加するための作業を実行して
```

---

### インスタンス 3: ドキュメント更新再開

> 元プロンプト: *OK まずこのホムペの状況を踏まえて arc42を更新して*

**変数の値**
  - `作業対象`: プロジェクトのドキュメント
  - `作業内容`: 更新
  - `追加条件`: ホムペの状況を踏まえて

**実際のプロンプト**
```
OK まずこのホムペの状況を踏まえて プロジェクトのドキュメントを更新して
```
