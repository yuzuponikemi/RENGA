# 問題解決の提案スキル

**説明**: 課題に対して改善案や代替案を提案するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 8 件 | **ユニークユーザー**: 8 人 | **カテゴリ**: 問題解決の提案

---

## テンプレートプロンプト

```
以下の状況を踏まえて、{課題内容} に対して {目標} を達成するための具体的な提案をしてください。考慮すべき制約は {制約条件} です。また、{追加の考慮点} も含めて検討してください。
```

## 変数一覧

- `{{課題内容}}`: 解決したい問題や現状の課題（例: 業務効率化だけの評価軸）
- `{{目標}}`: 達成したい理想状態や目的（例: 社員の幸福感向上）
- `{{制約条件}}`: 考慮すべき制約や前提（例: 資本主義の枠組み）
- `{{追加の考慮点}}`: さらに検討すべき要素や視点（例: ウィンウィンの関係構築）

## 活用シーン

- 業務プロセスの改善案を考えるとき
- システム設計の方向性を検討するとき
- コミュニケーション戦略を立案するとき

## 具体インスタンス

### インスタンス 1: 資本主義への対抗案

> 元プロンプト: *しかし資本家には抗えないのか　このハッカソンそのものが資本家有利の形そのもので、私もそれに組み込まれている　評価軸は業務効率化だけだし　どうにかして抗いたいね　たとえば尊厳を守る、社員の幸福感や帰属意識をあげる、カルチャーをつくるような仕組*

**変数の値**
  - `課題内容`: ハッカソンが資本家有利の形で評価軸が業務効率化だけになっていること
  - `目標`: 資本家に抗い、社員の尊厳や幸福感を守る
  - `制約条件`: 資本主義の枠組みの中で活動している
  - `追加の考慮点`: 社員の帰属意識向上やカルチャー形成、作業の偏りのバランス

**実際のプロンプト**
```
以下の状況を踏まえて、ハッカソンが資本家有利の形で評価軸が業務効率化だけになっていること に対して 資本家に抗い、社員の尊厳や幸福感を守る ことを達成するための具体的な提案をしてください。考慮すべき制約は 資本主義の枠組みの中で活動している です。また、社員の帰属意識向上やカルチャー形成、作業の偏りのバランス も含めて検討してください。
```

### インスタンス 2: システム設計の判断

> 元プロンプト: *うーん　この{システム名}は、DeepResearchの仕組みを構築する練習として１からつくってみたものですが、　ここであらためて　あなたはこの構造を拡張すべきか、それとも１から設計しなおすほうがいいと思いますか？　最終的には、このような章*

**変数の値**
  - `課題内容`: {システム名}というDeepResearchの仕組みを構築する練習用システムの拡張か再設計か
  - `目標`: 最終的にFactチェックやエンジニアリング設計に使えるシステムを構築する
  - `制約条件`: 現在は書籍情報のWebからの再構築を課題としている
  - `追加の考慮点`: 章番号解決以外の情報精査機能

**実際のプロンプト**
```
以下の状況を踏まえて、{システム名}というDeepResearchの仕組みを構築する練習用システムの拡張か再設計か に対して 最終的にFactチェックやエンジニアリング設計に使えるシステムを構築する ことを達成するための具体的な提案をしてください。考慮すべき制約は 現在は書籍情報のWebからの再構築を課題としている です。また、章番号解決以外の情報精査機能 も含めて検討してください。
```

### インスタンス 3: メディア露出の戦略

> 元プロンプト: *うーん　それはおいておいて　こういうふうにメディアに自分を晒すなら　それ専用のgmailアドレス作っておいてそれだけ晒すのがいい気がしていますがどうでしょう　全部{サービス名}経由なのか　お問い合わせはgoogle formでとか将来拡張し*

**変数の値**
  - `課題内容`: メディアに自分を晒す際の連絡先管理
  - `目標`: 安全かつ拡張可能な形で情報を公開する
  - `制約条件`: 現在は{サービス名}経由で運用している
  - `追加の考慮点`: 将来のお問い合わせフォームやサブドメイン設計

**実際のプロンプト**
```
以下の状況を踏まえて、メディアに自分を晒す際の連絡先管理 に対して 安全かつ拡張可能な形で情報を公開する ことを達成するための具体的な提案をしてください。考慮すべき制約は 現在は{サービス名}経由で運用している です。また、将来のお問い合わせフォームやサブドメイン設計 も含めて検討してください。
```
