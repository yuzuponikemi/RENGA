# 学習内容の整理・解説

**説明**: 学んだ内容やシステム構成をわかりやすく解説するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **ユニークユーザー**: 3 人 | **カテゴリ**: 学習内容の整理・解説

---

## テンプレートプロンプト

```
学習のため、これまで学んできたことや {操作対象} 上での操作や作成した {リソース種別}、{データ管理方法}、設定している {接続先} など、それぞれなんなのか、なぜ必要なのか、何をしようとしているのかまとめたいです。{ドキュメントルール} にしたがって、記事にしてください。
```

## 変数一覧

- `{{操作対象}}`: 学習したプラットフォームやツール名（例: クラウドサービス, 開発ツール）
- `{{リソース種別}}`: 作成したリソースの種類（例: データベース, 仮想マシン, ストレージ）
- `{{データ管理方法}}`: データの管理方式（例: データベース, ファイルシステム, キャッシュ）
- `{{接続先}}`: 設定しているエンドポイントや接続先（例: エンドポイント, API, コンテナ）
- `{{ドキュメントルール}}`: 記事作成時のルールやテンプレート（例: 指定されたリポジトリのルール, 社内フォーマット）

## 活用シーン

- クラウドサービスの学習内容を整理して記事化する
- システムのアーキテクチャをチームに説明する資料を作る
- 新しいツールの使い方を自分用にまとめる
