## テンプレートプロンプト

```
{調査テーマ}について、{調査範囲}を確認し、{調査目的}の観点から{出力形式}でまとめてください。{追加指示}
```

## 変数一覧

- `{{調査テーマ}}`: 調査・要約したい具体的なテーマ（例: 特定の機能の設定方法）
- `{{調査範囲}}`: 調査対象となる情報源やフォルダ（例: 指定したフォルダ）
- `{{調査目的}}`: なぜ調査するのか、何を明らかにしたいか（例: 特定の環境での設定方法の理解）
- `{{出力形式}}`: 出力してほしい形式（例: 日本語のドキュメント、技術記事）
- `{{追加指示}}`: その他の条件や要望（例: 具体的な手順を記載、複数記事に分けてOK）

## 活用シーン

- 特定の機能の設定方法を調査してマニュアル化したいとき
- 過去の失敗事例を分析して原因をまとめたいとき
- 複数の情報源から論点を抽出して記事にしたいとき

## 具体インスタンス

### インスタンス 1: cronジョブ設定調査

> 元プロンプト: *これはcc-traceという機能で作成したんですけど、cronにいれたりしないとやはり古いものしか残らないですよね　macbookでCron登録するのってどうやればいいかな　cc-traceフォルダも確認して、やり方をまとめて*

**変数の値**
  - `調査テーマ`: cronジョブの設定方法
  - `調査範囲`: 指定したフォルダ
  - `調査目的`: 特定の環境でCron登録する方法
  - `出力形式`: 日本語のドキュメント
  - `追加指示`: 古いものしか残らない問題も考慮して、具体的な手順を記載してください

**実際のプロンプト**
```
cronジョブの設定方法について、{調査範囲}を確認し、{調査目的}の観点から日本語のドキュメントにまとめてください。古いものしか残らない問題も考慮して、具体的な手順を記載してください。
```

### インスタンス 2: 技術記事作成依頼

> 元プロンプト: *今回いろいろなやんでうまくいったり行かなかったところを技術記事にしましょう　モデル選びから、論点を抽出し、文字数を制限したり、ハルシネーション対策で確認入れたり、紆余曲折ありました　それらすべてをまとめて最終結果がこれである　こういうストー*

**変数の値**
  - `調査テーマ`: モデル選びから論点抽出、文字数制限、ハルシネーション対策までの紆余曲折
  - `調査範囲`: 指定したフォルダ
  - `調査目的`: 技術記事としてまとめる
  - `出力形式`: 複数の記事に分けて
  - `追加指示`: 最終結果がこれであるというストーリーで、長くなっても大丈夫です

**実際のプロンプト**
```
モデル選びから論点抽出、文字数制限、ハルシネーション対策までの紆余曲折について、{調査範囲}を確認し、技術記事としてまとめる観点から複数の記事に分けてまとめてください。最終結果がこれであるというストーリーで、長くなっても大丈夫です。
```
