# 🔷 スキル定義: 意図確認と解釈のスキル

**説明**: ユーザーの曖昧な意図を確認・解釈し、適切な方向性を提案する
**ステータス**: ✅ 公開中 | **ソース数**: 18 件 | **カテゴリ**: 意図の確認と解釈

---

## テンプレートプロンプト（抽象）

```
あなたは{役割}です。以下のユーザーの発言から、その真の意図や背景を解釈し、確認してください。

ユーザーの発言: 「{ユーザー発言}」

解釈の観点:
1. ユーザーが暗に求めていること（{暗黙の要求}）
2. 現在の状況や前提（{状況前提}）
3. 確認すべきポイント（{確認ポイント}）

上記を踏まえ、以下の形式で応答してください:
- 意図の解釈: {解釈内容}
- 確認質問: {確認質問}
- 提案: {提案内容}
```

## 変数一覧

- `{{役割}}`: あなたが果たすべき役割（例: AIアシスタント、プロジェクトマネージャー）
- `{{ユーザー発言}}`: ユーザーが実際に発した言葉（例: 「できたと思いますが、エンドポイントとかどこに書いてるんですかね」）
- `{{暗黙の要求}}`: ユーザーが直接言っていないが求めていること（例: エンドポイントの場所を教えてほしい）
- `{{状況前提}}`: 会話の背景や前提条件（例: 開発中のプロジェクトでAPI設計を議論中）
- `{{確認ポイント}}`: ユーザーに確認すべき具体的な項目（例: エンドポイントのドキュメントは存在するか）
- `{{解釈内容}}`: ユーザーの意図を解釈した文章（例: ユーザーは実装が完了したが、エンドポイントの所在が不明で困っている）
- `{{確認質問}}`: ユーザーに投げかける確認の質問（例: 「エンドポイントのドキュメントは別ファイルにありますか？」）
- `{{提案内容}}`: 解釈に基づく具体的な提案（例: 「docs/endpoints.md を確認してみましょう」）

## 活用シーン

- プロジェクトの要件が曖昧な場合の意図確認
- ユーザーの質問の背景を理解して適切に回答する
- 会話の文脈を踏まえた提案や調整を行う
