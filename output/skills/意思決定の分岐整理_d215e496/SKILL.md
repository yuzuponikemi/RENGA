# 意思決定の分岐整理

**説明**: 複数の選択肢を比較し、条件に応じた意思決定の分岐を整理するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **ユニークユーザー**: 4 人 | **カテゴリ**: 意思決定のための分岐整理

---

## テンプレートプロンプト

```
以下の状況で、{選択肢A} と {選択肢B} を比較し、{判断基準} に基づいて最適な意思決定の分岐を整理してください。

状況: {状況説明}

条件:
- {条件1}
- {条件2}
- {条件3}

期待する出力:
1. 各選択肢のメリット・デメリット
2. 条件に応じた分岐ロジック（if-then形式）
3. 推奨される判断フロー
```

## 変数一覧

- `{{選択肢A}}`: 比較する最初の選択肢（例: モデルAを使う）
- `{{選択肢B}}`: 比較する2つ目の選択肢（例: モデルBを使う）
- `{{判断基準}}`: 意思決定の基準（例: 精度、コスト、時間）
- `{{状況説明}}`: 現在の状況や背景（例: ファクトチェックの点数が低い）
- `{{条件1}}`: 分岐の条件1（例: タイムアウト時間が短い場合）
- `{{条件2}}`: 分岐の条件2（例: クレーム数が少ない場合）
- `{{条件3}}`: 分岐の条件3（例: 再生成が必要な場合）

## 活用シーン

- AIモデル選定の意思決定分岐を整理する
- プロジェクトの進め方に関する複数案を比較検討する
- トラブルシューティングの条件分岐フローを作成する

## 具体インスタンス

### インスタンス 1: モデル選定の分岐

> 元プロンプト: *{モデルA}も入れてあるので、それで試して見てほしい　それがダメならタイムアウト時間をより長くしてでも{モデルB}にやらせましょう　クレーム数が{数値}というのも数が少なくて違和感ありますね*

**変数の値**
  - `選択肢A`: {モデルA}
  - `選択肢B`: {モデルB}
  - `判断基準`: 精度とタイムアウト時間
  - `状況説明`: ファクトチェックの点数が低い問題を解決するため、適切なモデルを選定したい
  - `条件1`: {モデルA}が利用可能である
  - `条件2`: {モデルB}はタイムアウト時間を長く設定できる
  - `条件3`: クレーム数が{数値}と少なく違和感がある

**実際のプロンプト**
```
以下の状況で、{モデルA} と {モデルB} を比較し、精度とタイムアウト時間 に基づいて最適な意思決定の分岐を整理してください。

状況: ファクトチェックの点数が低い問題を解決するため、適切なモデルを選定したい

条件:
- {モデルA}が利用可能である
- {モデルB}はタイムアウト時間を長く設定できる
- クレーム数が{数値}と少なく違和感がある

期待する出力:
1. 各選択肢のメリット・デメリット
2. 条件に応じた分岐ロジック（if-then形式）
3. 推奨される判断フロー
```

### インスタンス 2: ファクトチェック改善分岐

> 元プロンプト: *ごめん、再生成は途中でもいいのでここでストップしましょう　なぜファクトチェックの結果が点数として低いのか、その原因を特定して　生成側の仕組みやプロンプトを工夫してやり直して見ましょう*

**変数の値**
  - `選択肢A`: 再生成を途中で止める
  - `選択肢B`: 原因特定後に再生成する
  - `判断基準`: 品質改善の効率
  - `状況説明`: ファクトチェックの結果が点数として低い原因を特定し、生成側の仕組みやプロンプトを工夫してやり直したい
  - `条件1`: 再生成は途中でもストップ可能である
  - `条件2`: 原因特定が先に必要である
  - `条件3`: 生成側の仕組みやプロンプトを工夫する必要がある

**実際のプロンプト**
```
以下の状況で、再生成を途中で止める と 原因特定後に再生成する を比較し、品質改善の効率 に基づいて最適な意思決定の分岐を整理してください。

状況: ファクトチェックの結果が点数として低い原因を特定し、生成側の仕組みやプロンプトを工夫してやり直したい

条件:
- 再生成は途中でもストップ可能である
- 原因特定が先に必要である
- 生成側の仕組みやプロンプトを工夫する必要がある

期待する出力:
1. 各選択肢のメリット・デメリット
2. 条件に応じた分岐ロジック（if-then形式）
3. 推奨される判断フロー
```
