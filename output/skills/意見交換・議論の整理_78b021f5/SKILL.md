# 意見交換・議論の整理

**説明**: 複数の意見や議論を整理し、建設的な方向に導くスキル
**ステータス**: ✅ 公開中 | **ソース数**: 8 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 意見交換・議論

---

## テンプレートプロンプト

```
以下の議論や意見について、{議論のテーマ} に関して {議論の目的} を達成するために整理してください。特に {注目すべき観点} に注意し、{考慮すべき制約} を踏まえた上で、{期待する出力形式} でまとめてください。
```

## 変数一覧

- `{{議論のテーマ}}`: 議論の対象となっているトピックや問題（例: 特定の技術の公開の是非、特定のデータの処理方法）
- `{{議論の目的}}`: 議論を通じて達成したいこと（例: 合意形成、課題の明確化、次のアクション決定）
- `{{注目すべき観点}}`: 特に重視すべき視点や論点（例: 法的リスク、技術的実現可能性、ユーザー体験）
- `{{考慮すべき制約}}`: 議論の前提となる制約条件（例: 組織のポリシー、リソース制限、時間的制約）
- `{{期待する出力形式}}`: 整理結果の出力形式（例: 箇条書きリスト、フローチャート、比較表）

## 活用シーン

- チーム内の技術選定に関する議論を整理する
- プロジェクトの課題に対する複数の解決策を比較検討する
- 会議での意見を構造化して次のアクションを決める

## 具体インスタンス

### インスタンス 1: OSS公開の是非議論

> 元プロンプト: *であれば、最終的にはこれはOSSとして公開されるべきですね　実際の処理はローカルLLMで時折やればいい程度の分量な気もしますし　そして使うのは特定の企業のインフラではなくブロックチェーンか　でもそうなると、ずる賢い人間はその連帯をすり抜けて*

**変数の値**
  - `議論のテーマ`: OSS公開の是非
  - `議論の目的`: 合意形成
  - `注目すべき観点`: 法的リスクと技術的実現可能性
  - `考慮すべき制約`: 組織のポリシーとリソース制限
  - `期待する出力形式`: 賛成意見・反対意見・妥協案の3つのカテゴリに分類した箇条書きリスト

**実際のプロンプト**
```
以下の議論や意見について、{議論のテーマ} に関して 合意形成 を達成するために整理してください。特に 法的リスクと技術的実現可能性 に注意し、組織のポリシーとリソース制限 を踏まえた上で、賛成意見・反対意見・妥協案の3つのカテゴリに分類した箇条書きリスト でまとめてください。
```

### インスタンス 2: 音声認識タグ付け議論

> 元プロンプト: *そうですね〜　いったん特定の音声コンテンツの話題に戻りますか　チャンネルとかゲスト名を取っても、会話なのでそれがどっちの発言なのかってトランスクリプトには書いてないんですよね　まず音声認識で最初にタグ付けするしかない気がしますね　できるかな*

**変数の値**
  - `議論のテーマ`: 特定の音声コンテンツのトランスクリプトにおける発言者タグ付け
  - `議論の目的`: 実装方法の明確化
  - `注目すべき観点`: 音声認識の精度と処理フロー
  - `考慮すべき制約`: 既存の音声認識APIの制約とコスト
  - `期待する出力形式`: 処理手順をステップごとに説明したフローチャート形式

**実際のプロンプト**
```
以下の議論や意見について、特定の音声コンテンツのトランスクリプトにおける発言者タグ付け に関して 実装方法の明確化 を達成するために整理してください。特に 音声認識の精度と処理フロー に注意し、既存の音声認識APIの制約とコスト を踏まえた上で、処理手順をステップごとに説明したフローチャート形式 でまとめてください。
```
