# 🔷 スキル定義: 技術理解の整理・解説

**説明**: 学習内容やシステム構成を整理し、わかりやすく解説するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **カテゴリ**: 技術理解の整理・解説

---

## テンプレートプロンプト（抽象）

```
学習のため、これまで学んできたことや {対象システム} 上での操作や作成したリソース、{リソース種別}、設定している {設定項目} など、それぞれなんなのか、なぜ必要なのか、何をしようとしているのかまとめたいです。{ドキュメント規約} にしたがって、記事にしてください。
```

## 変数一覧

- `{{対象システム}}`: 操作対象のシステム名やプラットフォーム（例: クラウドサービス, フレームワーク, ツール）
- `{{リソース種別}}`: 作成・管理しているリソースの種類（例: データベース, コンテナ, パラメータグリッド）
- `{{設定項目}}`: 設定しているエンドポイントやパラメータ名（例: エンドポイント, 辞書のキー, マウント設定）
- `{{ドキュメント規約}}`: 記事作成時のフォーマットやルール（例: 社内Wikiの形式, リポジトリのしきたり）

## 活用シーン

- クラウドサービスの学習内容をブログ記事にまとめる
- システム構成を新メンバー向けに解説するドキュメントを作成する
- ツールの動作フローを図解入りで説明する資料を作る
