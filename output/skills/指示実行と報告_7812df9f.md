# 🔷 スキル定義: 指示実行と報告

**説明**: ユーザーの指示に従い作業を実行し、結果を報告するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **カテゴリ**: 指示実行と報告

---

## テンプレートプロンプト（抽象）

```
{作業内容}を実行してください。具体的には{具体的な手順}を行い、{確認事項}を確認した上で、{報告方法}で報告してください。
```

## 変数一覧

- `{{作業内容}}`: 実行すべき作業の概要（例: ページの修正と動作確認）
- `{{具体的な手順}}`: 作業の詳細な手順や条件（例: エラーを修正し、ブラウザで開いて確認）
- `{{確認事項}}`: 作業完了前に確認すべき項目（例: 正常に動作すること）
- `{{報告方法}}`: 結果の報告方法や形式（例: メールで報告、チャットで報告）

## 活用シーン

- Webページのエラー修正と動作確認の依頼
- 配信設定の確認と報告
- 記事生成と公開までの一連の作業依頼

# 🔶 具体インスタンス

### インスタンス 1: 404エラー修正と報告

> 元プロンプト: *404 not foundになっているかも　まず修正して、開いて動作確認までやってから報告お願いします*

**変数の値**
  - `作業内容`: エラーになっているページの修正
  - `具体的な手順`: 該当ページのリンクやパスを修正
  - `確認事項`: ブラウザで開いて正常に表示されること
  - `報告方法`: チャットで修正完了を報告

**実際のプロンプト**
```
エラーになっているページの修正を実行してください。具体的には該当ページのリンクやパスを修正し、ブラウザで開いて正常に表示されることを確認した上で、修正完了をチャットで報告してください。
```

---

### インスタンス 2: ニュースレター設定確認

> 元プロンプト: *そもそもなんですけど　どの記事をどうニュースレター化するのかってどういうふうに決めるの？それはsubstack側の設定？あとニュースレター化するなら記事はsubstack側に書く必要があるの？　ニュースレターでメール送信して、私が今週更新し*

**変数の値**
  - `作業内容`: 配信の設定確認
  - `具体的な手順`: どの記事を配信するかの決定方法、配信ツール側の設定確認、記事を配信ツールに書く必要があるかの確認、指定した期間に更新した記事一覧とリンクリストのみをメール送信する方法を調査
  - `確認事項`: 設定方法と実現可能性
  - `報告方法`: チャットで調査結果を報告

**実際のプロンプト**
```
配信の設定確認を実行してください。具体的にはどの記事を配信するかの決定方法、配信ツール側の設定確認、記事を配信ツールに書く必要があるかの確認、指定した期間に更新した記事一覧とリンクリストのみをメール送信する方法を調査し、結果をチャットで報告してください。
```

---

### インスタンス 3: 記事生成とプッシュ

> 元プロンプト: *おっけーです　またダリオ記事を生成、生成条件を付記してメインにプッシュまでお願いします*

**変数の値**
  - `作業内容`: 特定の人物に関する記事の生成
  - `具体的な手順`: 生成条件を付記して記事を生成し、メインにプッシュ
  - `確認事項`: 記事が正しく生成されプッシュされたこと
  - `報告方法`: チャットで完了を報告

**実際のプロンプト**
```
特定の人物に関する記事の生成を実行してください。具体的には生成条件を付記して記事を生成し、メインにプッシュまで行った上で、完了をチャットで報告してください。
```
