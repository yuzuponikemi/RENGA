# 🔷 スキル定義: 指示実行と改善

**説明**: ユーザーの指示を実行し、結果を確認して改善する
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **カテゴリ**: 指示実行と改善

---

## テンプレートプロンプト（抽象）

```
{指示内容}を実行してください。その後、{確認項目}を確認し、{改善方針}に基づいて改善を行ってください。
```

## 変数一覧

- `{{指示内容}}`: 実行してほしい具体的な作業内容（例: ツールのテスト、記事の調査）
- `{{確認項目}}`: 実行後に確認すべきポイント（例: 動作確認、結果の評価）
- `{{改善方針}}`: 改善の方向性や具体的な方法（例: リトライ処理の追加、比較表の作成）

## 活用シーン

- ツールのセットアップ後に動作確認と改善を依頼する
- 記事の調査結果を基に情報追加と比較分析を依頼する
- プログラムのエラー検知とリトライ機能の追加を依頼する

# 🔶 具体インスタンス

**変数の値**
  - `指示内容`: {システム名}のキーをセットしたので、テスト

**実際のプロンプト**
```
{システム名}のキーをセットしたので、テストしてみてください。うまく動いていれば、メインの{プロジェクト名}の方を動作確認、改善してみてください。
```

### インスタンス 2: 記事調査と比較改善

> 元プロンプト: *今回の記事はオープンジョー優先みたいでしたけど、たとえばcdg往復、ロンドン往復に加えてユーロスター往復のパターンはどうでしょうか　これらについても調査して、記事に情報を加えて、最後にすべてを比較するかたちで考えてください　価格も重要ですが*

**変数の値**
  - `指示内容`: {都市A}往復、{都市B}往復、{交通手段}往復のパターンを調査
  - `確認項目`: 価格と直行便の有無、{地域の紛争リスク}での足止めリスク
  - `改善方針`: 記事に情報を追加し、最後にすべてを比較する形に改善

**実際のプロンプト**
```
今回の記事は{特定の条件}優先みたいでしたけど、たとえば{都市A}往復、{都市B}往復に加えて{交通手段}往復のパターンはどうでしょうか。これらについても調査して、記事に情報を加えて、最後にすべてを比較するかたちで考えてください。価格も重要ですが、どちらかというと直行便で、{地域の紛争リスク}などの足止めリスクなどの回避を評価します。
```
