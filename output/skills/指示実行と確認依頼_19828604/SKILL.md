# 指示実行と確認依頼

**説明**: AIに作業指示を出し、結果確認や改善を依頼するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 10 件 | **ユニークユーザー**: 6 人 | **カテゴリ**: 指示実行と確認依頼

---

## テンプレートプロンプト

```
{作業対象}で{具体的な作業}を実行し、{確認ポイント}を確認してください。{追加条件}があれば対応し、{最終アクション}まで完了してください。
```

## 変数一覧

- `{{作業対象}}`: 作業を行う対象（例: 特定のシステム、APIキー、記事、アプリ）
- `{{具体的な作業}}`: 実行する具体的な作業内容（例: テスト実行、生成、追加、削除）
- `{{確認ポイント}}`: 確認すべき項目（例: 出力内容、動作確認、壊れていないか）
- `{{追加条件}}`: 追加で満たすべき条件（例: 速度設定、内容の充実、特定のファイルを参照）
- `{{最終アクション}}`: 完了後のアクション（例: プッシュ、記録、報告）

## 活用シーン

- AIにコード生成を依頼し、結果を確認して修正する
- AIに記事追加を依頼し、サイトが壊れていないか確認する
- AIに設定変更を依頼し、動作確認してからプッシュする

## トリガーフレーズ

- `これやってみて、確認してください`
- `テストしてみて、動作確認して`
- `生成し直してみて、確認して`
- `追加して、壊れていないか確認して`
- `削除して、すすめてください`

## 具体インスタンス

### インスタンス 1: デカルト作業確認

> 元プロンプト: *OK 今回方法序説の本の解説であることが会話からわかりづらいです。またEpisode1なのに、前回何について語ったかが入っていて矛盾なきはします。判明した課題についても取り組んでみて、もう一度生成し直してみてください*

**変数の値**
  - `作業対象`: {システム名}
  - `具体的な作業`: {書籍名}の解説を生成
  - `確認ポイント`: 出力内容、会話から本の解説とわかるか、矛盾の有無
  - `追加条件`: 会話から本の解説であることを明確にする、{エピソード番号}の矛盾を修正、判明した課題に対応
  - `最終アクション`: 生成し直して報告

**実際のプロンプト**
```
{システム名}で{書籍名}の解説を生成し、出力を確認してください。会話から本の解説であることがわかりづらいので、明確にしてください。また{エピソード番号}なのに前回の話題が入っている矛盾を修正し、判明した課題にも取り組んで、もう一度生成し直してみてください。
```

### インスタンス 2: 記事追加と確認

> 元プロンプト: *それで、前回追加した記事についてmkdocs.ymlに追加されていないので、webサイトが壊れています　これは追加してください　また同じようにwebサイトが壊れていないか確認する方法を考え、 それを含む記事の追加方法をmdに記録しておいてく*

**変数の値**
  - `作業対象`: 前回追加した記事
  - `具体的な作業`: {設定ファイル名}に追加、webサイトの確認方法を考案
  - `確認ポイント`: webサイトが壊れていないか
  - `追加条件`: 確認方法を{記録ファイル名}に記録、記事追加依頼時に{記録ファイル名}を読む仕組みを作る
  - `最終アクション`: {記録ファイル名}に記録して完了

**実際のプロンプト**
```
前回追加した記事について{設定ファイル名}に追加し、webサイトが壊れていないか確認してください。また同じようにwebサイトが壊れていないか確認する方法を考え、それを含む記事の追加方法を{記録ファイル名}に記録しておいてください。そして記事の追加を依頼されたときは必ずその{記録ファイル名}を読むようにしてください。
```

### インスタンス 3: モデル削除と差し替え

> 元プロンプト: *qwen3-next は大きすぎてもう使わないので削除しておいて、そしてそのあとGemma 4プルしてすすめてください*

**変数の値**
  - `作業対象`: {削除対象モデル名}
  - `具体的な作業`: 削除、{追加モデル名}のプル
  - `確認ポイント`: 削除完了、プル完了
  - `追加条件`: {削除対象モデル名}は大きすぎて使わないため削除
  - `最終アクション`: プル後にすすめて報告

**実際のプロンプト**
```
{削除対象モデル名} は大きすぎてもう使わないので削除し、そのあと{追加モデル名}をプルしてすすめてください。
```
