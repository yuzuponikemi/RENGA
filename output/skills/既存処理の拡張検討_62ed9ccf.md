# 🔷 スキル定義: Podcast処理拡張設計

**説明**: 既存Podcastパイプラインの拡張・自動化・統合を検討するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 12 件 | **カテゴリ**: 既存処理の拡張検討

---

## テンプレートプロンプト（抽象）

```
以下の既存処理を拡張したい。

【現状】
- {既存パイプライン名} で {処理対象} を {現在の処理内容} している
- 出力先: {現在の出力先}
- 実行頻度: {現在の実行頻度}

【拡張したい方向性】
- {拡張したい機能1}: {機能1の詳細}
- {拡張したい機能2}: {機能2の詳細}
- {統合対象}: {統合したい既存システムやデータ}

【制約・注意点】
- {制約条件1}
- {制約条件2}

上記を踏まえて、{出力形式} で設計・計画を提案してください。
```

## 変数一覧

- `{{既存パイプライン名}}`: 現在稼働中のパイプライン名（例: 既存パイプライン）
- `{{処理対象}}`: 処理しているデータの種類（例: 音声データ）
- `{{現在の処理内容}}`: 現在の処理の概要（例: 文字起こし→要約→記事化）
- `{{現在の出力先}}`: 成果物の保存先（例: 指定のリポジトリ）
- `{{現在の実行頻度}}`: 現在の実行スケジュール（例: 手動実行）
- `{{拡張したい機能1}}`: 追加したい機能の名前（例: 自動ダウンロード）
- `{{機能1の詳細}}`: 機能1の具体的な要件（例: 新着データを自動取得）
- `{{拡張したい機能2}}`: 追加したい機能の名前（例: フォーマット統一）
- `{{機能2の詳細}}`: 機能2の具体的な要件（例: 全データのフォーマットを統一）
- `{{統合対象}}`: 統合したい既存システムやデータソース（例: 既存アーカイブ）
- `{{制約条件1}}`: 技術的制約や優先順位（例: まず基本機能だけ整備）
- `{{制約条件2}}`: その他の制約（例: 実行スケジュールは毎日深夜）
- `{{出力形式}}`: 期待する出力形式（例: 設計ドキュメント）

## 活用シーン

- 既存のデータ処理パイプラインを自動化・拡張するとき
- 複数のデータソースを統合した一元管理システムを設計するとき
- 既存のバッチ処理を定期実行に移行するとき

  - `機能1の詳細`: 新着データを指定の配信元から自動取得して文字起こし

  - `制約条件2`: cronは毎日深夜3時JST一回実行

### インスタンス 3: 複数リポジトリ統合設計

> 元プロンプト: *私他のプロジェクトで　複数のコンテンツ生成システムをいろんなリポジトリでたくさんコンテンツ生成してまして*

**変数の値**
  - `既存パイプライン名`: 複数リポジトリ
  - `処理対象`: 複数種類のコンテンツ
  - `現在の処理内容`: 個別にコンテンツ生成
  - `現在の出力先`: 各リポジトリの特定フォルダ
  - `現在の実行頻度`: 手動
  - `拡張したい機能1`: 一元管理
  - `機能1の詳細`: 全リポジトリのコンテンツを指定のリポジトリに集約
  - `拡張したい機能2`: 自動同期
  - `機能2の詳細`: ワークフローで特定フォルダの内容を指定のリポジトリにPR作成
  - `統合対象`: 複数のコンテンツ生成システム
  - `制約条件1`: 既存のアーカイバのcron出力を全て指定のリポジトリにPR作成する形に変更
  - `制約条件2`: まずは特定のnotebookをルートに置いてテスト
  - `出力形式`: 設計ドキュメント

**実際のプロンプト**
```
以下の既存処理を拡張したい。

【現状】
- 複数リポジトリ で 複数種類のコンテンツ を 個別にコンテンツ生成 している
- 出力先: 各リポジトリの特定フォルダ
- 実行頻度: 手動

【拡張したい方向性】
- 一元管理: 全リポジトリのコンテンツを指定のリポジトリに集約
- 自動同期: ワークフローで特定フォルダの内容を指定のリポジトリにPR作成
- 統合対象: 複数のコンテンツ生成システム

【制約・注意点】
- 既存のアーカイバのcron出力を全て指定のリポジトリにPR作成する形に変更
- まずは特定のnotebookをルートに置いてテスト

上記を踏まえて、設計ドキュメント で設計・計画を提案してください。
```
