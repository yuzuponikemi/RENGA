# 🔷 スキル定義: 機能改善・拡張指示

**説明**: 既存機能の改善や新機能追加を指示するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **カテゴリ**: 機能改善・拡張指示

---

## テンプレートプロンプト（抽象）

```
{対象システム}において、{改善内容}を実施してください。具体的には{具体的な実装方法}の方向性で進め、{テスト要件}を満たすようにしてください。また、{追加機能}も併せて実装し、{期待効果}を達成したいと考えています。
```

## 変数一覧

- `{{対象システム}}`: 改善対象のシステム名や機能名（例: 検索機能、コンテンツ生成パイプライン）
- `{{改善内容}}`: 具体的に何を改善・拡張するか（例: 検索の問題を修正、構成の精度向上）
- `{{具体的な実装方法}}`: 技術的な実装方針やアプローチ（例: 特定の検索ツールを使う、複数言語で検索）
- `{{テスト要件}}`: テストの条件や確認すべきポイント（例: 個別にテスト可能、複数のデータで動作確認）
- `{{追加機能}}`: 新たに追加したい機能（例: 詳細な中間ファイル作成、差分更新対応）
- `{{期待効果}}`: 改善によって得たい効果（例: 学習効率向上、パイプラインの正確性向上）

## 活用シーン

- 既存の検索機能に新しい検索エンジンを追加する際の指示
- コンテンツ生成パイプラインの精度を向上させるための改善依頼
- システムの更新に伴うテストコードやドキュメントのメンテナンス指示

# 🔶 具体インスタンス

### インスタンス 1: 検索機能改善＋読み物追加

> 元プロンプト: *検索の問題はすぐに修正お願いします　基本的には特定の検索ツールを使う方向性で、環境変数セットアップも使えるようにしておいてください　両方の検索が機能するように、個別にテストできるようにしましょう　またこのシステムにはポッドキャストの元にする*

**変数の値**
  - `対象システム`: 検索システム
  - `改善内容`: 検索の問題を修正
  - `具体的な実装方法`: 特定の検索ツールを使う方向性で、環境変数セットアップも使えるようにする
  - `テスト要件`: 両方の検索が機能するように、個別にテスト可能
  - `追加機能`: 特定のツールが出力するような詳細な読み物の作成機能
  - `期待効果`: 人間の学習がかなりやりやすくなる

**実際のプロンプト**
```
検索システムにおいて、検索の問題を修正してください。具体的には特定の検索ツールを使う方向性で、環境変数セットアップも使えるようにしておいてください。両方の検索が機能するように、個別にテストできるようにしましょう。また、このシステムにはポッドキャストの元にする中間ファイルとして、特定のツールが出力するような詳細な読み物も作成して欲しいです。その読み物があって、ポッドキャストがあると、人間の学習がかなりやりやすくなるかと思います。
```

### インスタンス 2: ブックガイド章構成改善

> 元プロンプト: *このコンテンツ生成のパイプラインが正しい章構成を見つけられるようにするべきです　そこだけ機能として抜き出して、複数の本でワークするようにテスト、改善できますか　たとえば複数のソースで同様の章立てを見つけたときにはじめてそれが書籍の章立てとし*

**変数の値**
  - `対象システム`: コンテンツ生成パイプライン
  - `改善内容`: 正しい章構成を見つけられるように改善
  - `具体的な実装方法`: 章構成検出機能だけを抜き出し、複数のソースで同様の章立てを見つけたときのみ成立と判定、英語・ロシア語・中国語で検索
  - `テスト要件`: 複数の本でワークするようにテスト・改善
  - `追加機能`: なし
  - `期待効果`: パイプラインの章構成精度向上

**実際のプロンプト**
```
コンテンツ生成パイプラインにおいて、正しい章構成を見つけられるように改善してください。具体的には章構成検出機能だけを抜き出して、複数の本でワークするようにテスト・改善してください。たとえば複数のソースで同様の章立てを見つけたときにはじめてそれが書籍の章立てとして成立すると判定する、英語やロシア語、中国語で検索して情報を集める、などの方法を取ってください。
```

インデックス更新システムにおいて、差分変更に対応するように改良してください。テストコードやドキュメントに更新が必要ないか確認し、必要なら変更してください。
