# 🔷 スキル定義: 現状確認と改善提案

**説明**: システムやモデルの現状を確認し、改善策を提案させるスキル
**ステータス**: ✅ 公開中 | **ソース数**: 3 件 | **カテゴリ**: 現状確認と改善提案

---

## テンプレートプロンプト（抽象）

```
{対象領域}について、{確認したい観点}を{確認方法}で確認し、{期待する成果}を得るために{改善アクション}を提案してください。
```

## 変数一覧

- `{{対象領域}}`: 確認・改善したい対象（例: エンティティと関係、特定の章、モデルの性能）
- `{{確認したい観点}}`: 何を確認したいか（例: 有用な知識が蓄積できているか、系列に拡張できるか、何が起きているか）
- `{{確認方法}}`: 確認の手段（例: フル情報で列挙、計画してみる、競わせて思考過程を見る）
- `{{期待する成果}}`: 確認後に得たい結果（例: 有用な知識の有無、拡張計画、問題の原因）
- `{{改善アクション}}`: 具体的な改善策（例: 列挙して確認、計画を立てる、問題を分析する）

## 活用シーン

- 既存のデータベースの品質チェックと改善点の洗い出し
- 機械学習モデルの性能評価とチューニング方針の立案
- ドキュメントや教材の内容拡張計画の作成

# 🔶 具体インスタンス

### インスタンス 1: エンティティ確認

> 元プロンプト: *まぁいいや　いったん忘れて　実際のエンティティや関係を１０個ほどフル情報で列挙してみてくれる？有用な知識が蓄積できているのか確認したいので*

**変数の値**
  - `対象領域`: 実際のエンティティや関係
  - `確認したい観点`: 有用な知識が蓄積できているか
  - `確認方法`: フル情報で列挙
  - `期待する成果`: 有用な知識の有無
  - `改善アクション`: 列挙して確認

**実際のプロンプト**
```
実際のエンティティや関係について、有用な知識が蓄積できているかをフル情報で列挙して確認し、有用な知識の有無を得るために列挙して確認してください。
```

> 元プロンプト: *neural-engineの章ですが、lstmとか、RNNとかの系列に拡張できそうですよね　計画してみてもらえますか*

**変数の値**
  - `対象領域`: 特定の章
  - `確認したい観点`: 系列に拡張できるか
  - `確認方法`: 計画してみる
  - `期待する成果`: 拡張計画
  - `改善アクション`: 計画を立てる

**実際のプロンプト**
```
特定の章について、系列に拡張できるかを計画してみることで確認し、拡張計画を得るために計画を立ててください。
```
