# 🔷 スキル定義: 環境セットアップ手順の作成

**説明**: 開発環境やツールのセットアップ手順を詳細に記述するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 8 件 | **カテゴリ**: 環境セットアップ手順

---

## テンプレートプロンプト（抽象）

```
# Role
あなたは熟練した{役割}です。
現在、{プロジェクト名}において{現状説明}。
ここに、{導入したいもの}を導入し、{達成したいこと}を実現したいです。

# Goal
{学習目的}を学習できるよう、
{既存資産}を活用して{作成するもの}を作成してください。

# Requirements
1. **セットアップ:**
   - {ツール名} をインストールするためのコマンドを教えてください。

2. **{作成物の種類}作成:**
   - {ファイルパス}を作成してください。
   - このファイル内で、{使用するライブラリ・関数}を使用してください。

3. **{実装内容の種類}:**
   - 初学者が理解しやすい以下の{項目数}つの{項目}を実装してください。
     - {項目1}: {説明1}（閾値: {閾値1}）
     - {項目2}: {説明2}（{補足条件}）

4. **{データの種類}:**
   - {データ取得方法}、{データ内容}を{データ数}件定義してください。
   - 例: {データ例}

5. **実行方法:**
   - ターミナルでこれを実行するためのコマンド（{コマンド例}等）を提示してください。

# Constraints
- {制約1}
- {制約2}
```

## 変数一覧

- `{{役割}}`: 担当者の専門性（例: QAエンジニア兼AIエンジニア）
- `{{プロジェクト名}}`: 対象プロジェクト名（例: プロジェクト名）
- `{{現状説明}}`: 現在の状態（例: トレース機能が実装されている）
- `{{導入したいもの}}`: 新たに導入する技術（例: 評価フレームワーク）
- `{{達成したいこと}}`: 導入で実現したいこと（例: 回答精度をテストできるようにする）
- `{{学習目的}}`: 学習者が習得したいこと（例: フレームワークの仕組み）
- `{{既存資産}}`: 既存のコードやリソース（例: 既存のメインロジック）
- `{{作成するもの}}`: 作成するファイルや成果物（例: 独立したテストファイル）
- `{{ツール名}}`: インストールするツール名（例: ツール名）
- `{{作成物の種類}}`: 作成するものの種類（例: テストファイル）
- `{{ファイルパス}}`: 作成するファイルのパス（例: tests/test_file.py）
- `{{使用するライブラリ・関数}}`: 使用するライブラリや関数（例: テスト用関数）
- `{{実装内容の種類}}`: 実装する内容のカテゴリ（例: 評価メトリクス）
- `{{項目数}}`: 実装する項目の数（例: 2）
- `{{項目}}`: 実装する項目の総称（例: メトリクス）
- `{{項目1}}`: 1つ目の項目名（例: メトリクス1）
- `{{説明1}}`: 1つ目の項目の説明（例: 回答が質問に関連しているか）
- `{{閾値1}}`: 1つ目の項目の閾値（例: 0.7）
- `{{項目2}}`: 2つ目の項目名（例: メトリクス2）
- `{{説明2}}`: 2つ目の項目の説明（例: 回答がコンテキストに基づいているか）
- `{{補足条件}}`: 2つ目の項目の補足条件（例: コンテキストなしでOK）
- `{{データの種類}}`: 使用するデータの種類（例: テストデータ）
- `{{データ取得方法}}`: データの取得方法（例: コード内にハードコード）
- `{{データ内容}}`: データの内容説明（例: ダミーの入力と期待される出力）
- `{{データ数}}`: 定義するデータの件数（例: 2）
- `{{データ例}}`: データの具体例（例: 質問と回答のペア）
- `{{コマンド例}}`: 実行コマンドの例（例: ツール test run ...）
- `{{制約1}}`: 1つ目の制約（例: 既存のソースコードは変更しない）
- `{{制約2}}`: 2つ目の制約（例: テストコードは既存の関数を正しく呼び出せるように書く）

## 活用シーン

- 新しい評価フレームワークを導入する際のテストコード作成手順
- 機械学習モデルの学習データ生成スクリプト作成手順
- 開発環境に特定のツールをセットアップする手順書作成

### インスタンス 2: 五目並べ学習データ生成

> 元プロンプト: *# Phase 2-1: 学習データ生成スクリプトの作成  現在、{プロジェクト名}が完成しています。 次のステップである「{達成したいこと}」のために、{導入したいもの}を作成してください。*

**変数の値**
  - `役割`: AIエンジニア
  - `プロジェクト名`: プロジェクト名
  - `現状説明`: アルゴリズムによるAIが完成しています
  - `導入したいもの`: 学習データ生成スクリプト
  - `達成したいこと`: モデルへの模倣学習のためのデータ準備
  - `学習目的`: プレイデータを大量に生成・保存する方法
  - `既存資産`: 既存のソースコード
  - `作成するもの`: 学習データ生成スクリプト
  - `ツール名`: Python（並列処理ライブラリ）
  - `作成物の種類`: スクリプト
  - `ファイルパス`: generate_data.py
  - `使用するライブラリ・関数`: 並列処理ライブラリ
  - `実装内容の種類`: 対戦形式とゆらぎの導入
  - `項目数`: 2
  - `項目`: 対戦形式
  - `項目1`: AI vs AI
  - `説明1`: 双方が最善手を打ち合う
  - `閾値1`: N/A
  - `項目2`: AI vs ランダムプレイヤー
  - `説明2`: 多様な盤面を学習させるため
  - `補足条件`: これらをランダムに混ぜて対戦させる
  - `データの種類`: 保存フォーマット
  - `データ取得方法`: スクリプト内で生成
  - `データ内容`: 各行に1局のデータ（着手位置のシーケンスと勝者）
  - `データ数`: 引数で指定（まずは100局程度）
  - `データ例`: {"input": [0, 1, 2, ...], "winner": 1}
  - `コマンド例`: python generate_data.py --games 100
  - `制約1`: 既存のソースコードを読み込んで使用する
  - `制約2`: 並列処理ライブラリを使用して並列処理する

**実際のプロンプト**
```
# Phase 2-1: 学習データ生成スクリプトの作成

現在、{プロジェクト名}が完成しています。
次のステップである「{達成したいこと}」のために、{導入したいもの}を作成してください。

## 要件
1. **スクリプト名**: {ファイルパス}
2. **対戦形式**:
   - {項目1}: {説明1}
   - {項目2}: {説明2}
   - これらをランダムに混ぜて対戦させてください。
3. **{補足条件}**:
   - 毎回同じ棋譜にならないよう、着手決定時にランダム性を少し入れてください。
4. **高速化（必須）**:
   - {使用するライブラリ・関数} を使用して、CPUコアをフルに使って並列で対戦させてください。
5. **保存フォーマット**:
   - {データ例}
   - 各行に1局のデータを保存。
   - フォーマット例: {データ例}
6. **目標**:
   - 引数で「生成する対戦数」を指定できるようにしてください。まずは{データ数}程度でテストします。

## 既存コードの参照
現在の {既存資産} を読み込んで使用してください。
```
