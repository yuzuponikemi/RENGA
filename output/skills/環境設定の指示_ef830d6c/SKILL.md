# 環境設定の指示

## テンプレートプロンプト

```
OK 私は常に{ツール種別}を使います　このワークスペースは{ワークスペース名}と名付けて作成しておいて　概要の追記もお願い
基本的に起動していますが、常に{プロキシ経由}で{ポート番号}を使うようにしています
いいですね　この{機能名}対応は当面は使いませんので、デフォルトは通常の{デフォルト動作}に戻しておいてもらえますか　特別に指定したばあいにのみこの{機能名}を使います
今回の変更で、{機能名}を使った{操作内容}が可能になったけど　変更内容を簡単にまとめて教えてくれますか　自分の狙いとしては、エージェントの{トリガー条件}のタイミングで　{選択肢A}するか　{選択肢B}を使うかどうかを切り替えれるようにしたかった
```

## 変数一覧

- `{{ツール種別}}`: 使用するツールの種類（例: CLI）
- `{{ワークスペース名}}`: 任意のワークスペース名
- `{{プロキシ経由}}`: プロキシの種類（例: 指定したプロキシ）
- `{{ポート番号}}`: 使用するポート番号（例: 指定したポート）
- `{{機能名}}`: 機能の名前（例: 特定の機能）
- `{{デフォルト動作}}`: デフォルトの動作（例: 通常の動作）
- `{{操作内容}}`: 実行する操作（例: 検索）
- `{{トリガー条件}}`: 切り替えのタイミング（例: 特定の条件）
- `{{選択肢A}}`: 切り替えの選択肢A（例: 動作A）
- `{{選択肢B}}`: 切り替えの選択肢B（例: 動作B）

## 活用シーン

- 新しいワークスペースを作成し、ツールのデフォルト設定を変更したいとき
- 特定の機能を一時的に無効にして、別の動作に切り替えたいとき
- 機能追加の変更内容を確認し、動作切り替えの仕組みを理解したいとき

## トリガーフレーズ

- `環境設定を変更して`
- `デフォルトを戻して`
- `設定を切り替えて`
- `ワークスペースを作成して`
- `変更内容をまとめて`
