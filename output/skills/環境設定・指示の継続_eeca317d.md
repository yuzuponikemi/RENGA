# 🔷 スキル定義: 環境設定・指示継続

**説明**: 設定変更や作業継続を指示する汎用テンプレート
**ステータス**: ✅ 公開中 | **ソース数**: 8 件 | **カテゴリ**: 環境設定・指示の継続

---

## テンプレートプロンプト（抽象）

```
{作業対象}について{具体的な指示}をしてください。{追加条件}に注意しながら、{次のアクション}を実行してください。
```

## 変数一覧

- `{{作業対象}}`: 設定変更や確認を行う対象（例: 環境変数、ファイル、システム設定）
- `{{具体的な指示}}`: 実行してほしい操作内容（例: 作成、修正、保存、再起動）
- `{{追加条件}}`: 注意点や制約事項（例: 他の著作の影響を受けないように、クリーンな状態で）
- `{{次のアクション}}`: 指示完了後の次のステップ（例: 次の手順を進める、コミットしてPushする）

## 活用シーン

- 開発環境の設定変更を依頼するとき
- ファイルの作成・修正を指示するとき
- システムの状態確認や再起動を依頼するとき

# 🔶 具体インスタンス

### インスタンス 1: 環境変数設定完了後の継続指示

> 元プロンプト: *とりあえず、個人としては　資本の圧力に負けないくらいスキルをみにつけて、対抗できるように備える　というところか　Step 1: 環境変数を設定がおわったので　ｓＴＥＰ２以降すすめてください*

**変数の値**
  - `作業対象`: 環境変数
  - `具体的な指示`: 設定が終わったので進める
  - `追加条件`: 個人として指定した方針に従う
  - `次のアクション`: 次の手順を進める

**実際のプロンプト**
```
環境変数を設定が終わったので、次の手順を進めてください。個人としては、指定した方針でお願いします。
```

### インスタンス 2: ファイル作成とクリーン管理指示

> 元プロンプト: *.env.exampleを作成してくれますか*

**変数の値**
  - `作業対象`: .env.example と gitignore
  - `具体的な指示`: 作成、整備、コミット、Push
  - `追加条件`: 変更が多くなっているのでクリーンな状態を保つ
  - `次のアクション`: 必要なものをコミットしてPushする

**実際のプロンプト**
```
.env.exampleを作成してください。また、変更が多くなっているのでgitignoreも合わせて整備し、必要なものはコミットしてPushしてクリーンにしておいてください。
```
