# 確認・再実行指示

**説明**: 過去の作業結果を確認し、必要に応じて再実行を依頼するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 確認・再実行指示

---

## テンプレートプロンプト

```
{確認対象}について、{確認内容}を確認したいので、{具体的なアクション}を実行してください。特に{注目ポイント}に注意して、{期待する結果}を確認できるようにしてください。
```

## 変数一覧

- `{{確認対象}}`: 確認したい対象（例: エンティティ一覧、ブックガイドのタイトル、実装コード）
- `{{確認内容}}`: 確認したい観点（例: 情報の正確性、章立ての誤り、品質比較）
- `{{具体的なアクション}}`: 実行してほしい作業（例: 列挙する、再処理する、実装する、比較用に作成する）
- `{{注目ポイント}}`: 特に注意すべき点（例: 有用な知識の蓄積、間違いの有無、生成条件の付記）
- `{{期待する結果}}`: 最終的に得たい成果（例: フル情報のリスト、修正済みデータ、品質比較可能な記事）

## 活用シーン

- 過去に生成したデータの正確性を再確認したいとき
- 実装結果の品質を以前のバージョンと比較したいとき
- 特定の処理が正しく行われているか検証したいとき

## トリガーフレーズ

- `確認して`
- `もう一度処理して`
- `比較したいので`
- `何が起きているか確認して`
- `間違っていないか確認`

## 具体インスタンス

### インスタンス 1: エンティティ確認依頼

> 元プロンプト: *まぁいいや　いったん忘れて　実際のエンティティや関係を{数値}個ほどフル情報で列挙してみてくれる？有用な知識が蓄積できているのか確認したいので*

**変数の値**
  - `確認対象`: 実際のエンティティや関係
  - `確認内容`: 情報の正確性
  - `具体的なアクション`: {数値}個ほどフル情報で列挙する
  - `注目ポイント`: 有用な知識が蓄積できているか
  - `期待する結果`: 確認できるようにする

**実際のプロンプト**
```
実際のエンティティや関係について、情報の正確性を確認したいので、{数値}個ほどフル情報で列挙してみてくれる？特に有用な知識が蓄積できているかに注意して、確認できるようにしてください。
```

---

### インスタンス 2: ブックガイド再処理

> 元プロンプト: *おけです　既存のブックガイドがあるのタイトル全てもう一度処理してみて、章立てで間違っているものがないか確認して*

**変数の値**
  - `確認対象`: 既存のブックガイドのタイトル
  - `確認内容`: 章立ての誤り
  - `具体的なアクション`: 全てもう一度処理する
  - `注目ポイント`: 章立てで間違っているものがないか
  - `期待する結果`: 修正済みデータを確認できるようにする

**実際のプロンプト**
```
既存のブックガイドのタイトルについて、章立ての誤りを確認したいので、全てもう一度処理してみて、間違っているものがないか確認して。特に章立てで間違っているものがないかに注意して、修正済みデータを確認できるようにしてください。
```

---

### インスタンス 3: 品質比較実装依頼

> 元プロンプト: *うん、これよさそうだね　これで実装して、比較用に作成してみて　あと、これまでと品質を比較したいので、今後ブログ上には　生成条件を付記した状態で記事を上げて*

**変数の値**
  - `確認対象`: これまでの実装
  - `確認内容`: 品質比較
  - `具体的なアクション`: 比較用に作成する
  - `注目ポイント`: 生成条件を付記した状態で記事を上げる
  - `期待する結果`: 品質比較可能な記事を確認できるようにする

**実際のプロンプト**
```
これまでの実装について、品質比較を確認したいので、比較用に作成してみて。特に生成条件を付記した状態で記事を上げることに注意して、品質比較可能な記事を確認できるようにしてください。
```
