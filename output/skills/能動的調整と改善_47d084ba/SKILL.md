## テンプレートプロンプト

```
以下の観点で、{対象作業}に対して能動的な調整と改善を行ってください。

1. 現状の課題: {現状の課題や問題点}
2. 改善の方向性: {希望する改善方向や目標状態}
3. 具体的な要件: {実現したい具体的な要件や制約条件}
4. 考慮すべき点: {考慮すべき追加の観点や注意点}

特に以下の点に注意して調整してください:
- {調整のポイント1}
- {調整のポイント2}
- {調整のポイント3}
```

## 変数一覧

- `{{対象作業}}`: 改善・調整の対象となる作業やプロジェクト（例: コンテンツ要約機能、ドキュメント生成）
- `{{現状の課題や問題点}}`: 現在の作業で認識している課題や不満点（例: 論点が少ない、結果しか見えない）
- `{{希望する改善方向や目標状態}}`: 理想的な状態や達成したい目標（例: 論点を増やす、逐次経過が確認できる）
- `{{実現したい具体的な要件や制約条件}}`: 実装上の具体的な要件や数値目標（例: 本論の長さを拡大、品質向上）
- `{{考慮すべき追加の観点や注意点}}`: 改善時に考慮すべき追加要素（例: 飽きさせない工夫、重複防止）
- `{{調整のポイント1}}`: 調整の際の重要な観点1（例: ユーザー体験の維持）
- `{{調整のポイント2}}`: 調整の際の重要な観点2（例: 技術的制約の考慮）
- `{{調整のポイント3}}`: 調整の際の重要な観点3（例: 将来の拡張性）

## 活用シーン

- コンテンツ要約の品質改善（論点追加・長さ調整）
- ドキュメント生成の構造改善（目次整理・重複防止）
- システムの可視性向上（逐次ログ出力・進捗確認）

## 具体インスタンス

### インスタンス 1: ポッドキャスト要約改善

> 元プロンプト: *でてきた2つの記事をよみました　いい感じなんだけど、2時間のポッドキャストに対しては論点が少ないですね　論点は10個以上キープさせて　本論の長さが3倍くらいになるようにできますか？それでいて前々回までのように、同じ構造の繰り返しの飽きやすい*

**変数の値**
  - `対象作業`: コンテンツ要約機能
  - `現状の課題や問題点`: 長時間のコンテンツに対して論点が少ない、本論の長さが不足している
  - `希望する改善方向や目標状態`: 論点を増やし、本論の長さを拡大し、飽きさせない読み物にする
  - `実現したい具体的な要件や制約条件`: 論点は指定した数以上キープ、本論の長さを指定倍に拡大、同じ構造の繰り返しにならない工夫
  - `考慮すべき追加の観点や注意点`: 過去の高品質な成果と同等の内容再現、エンティティ抽出の質向上
  - `調整のポイント1`: 飽きさせないための構造的工夫
  - `調整のポイント2`: エンティティ抽出の質を高める
  - `調整のポイント3`: 過去の高品質な成果を再現する

**実際のプロンプト**
```
以下の観点で、{対象作業}に対して能動的な調整と改善を行ってください。

1. 現状の課題: 長時間のコンテンツに対して論点が少ない、本論の長さが不足している
2. 改善の方向性: 論点を増やし、本論の長さを拡大し、飽きさせない読み物にする
3. 具体的な要件: 論点は指定した数以上キープ、本論の長さを指定倍に拡大、同じ構造の繰り返しにならない工夫
4. 考慮すべき点: 過去の高品質な成果と同等の内容再現、エンティティ抽出の質向上

特に以下の点に注意して調整してください:
- 飽きさせないための構造的工夫
- エンティティ抽出の質を高める
- 過去の高品質な成果を再現する
```

### インスタンス 2: シラバスドキュメント整理

> 元プロンプト: *ありがとう　今までにどんなトピックのどういう学習内容を網羅してきたのか、一目で確認できるような詳細なシラバスドキュメントを用意してもらえますかあとで何かコンテンツを追加したりするときに、重複していないかどうかなどの判断材料になるようにしたい*

**変数の値**
  - `対象作業`: 学習コンテンツのシラバスドキュメント
  - `現状の課題や問題点`: 古くなったカリキュラム文書が残っており、重複の判断が難しい
  - `希望する改善方向や目標状態`: 一目で確認できる詳細なシラバス、重複防止、更新の仕組み化
  - `実現したい具体的な要件や制約条件`: 全トピックと学習内容を網羅した詳細なシラバス、設定ファイルに更新ルールを記載
  - `考慮すべき追加の観点や注意点`: コンテンツ追加時の重複チェック、将来のアップデートの継続性
  - `調整のポイント1`: 一目で全容が把握できる構造
  - `調整のポイント2`: 重複防止のためのチェック機構
  - `調整のポイント3`: 設定ファイルへの更新ルールの明記

**実際のプロンプト**
```
以下の観点で、{対象作業}に対して能動的な調整と改善を行ってください。

1. 現状の課題: 古くなったカリキュラム文書が残っており、重複の判断が難しい
2. 改善の方向性: 一目で確認できる詳細なシラバス、重複防止、更新の仕組み化
3. 具体的な要件: 全トピックと学習内容を網羅した詳細なシラバス、設定ファイルに更新ルールを記載
4. 考慮すべき点: コンテンツ追加時の重複チェック、将来のアップデートの継続性

特に以下の点に注意して調整してください:
- 一目で全容が把握できる構造
- 重複防止のためのチェック機構
- 設定ファイルへの更新ルールの明記
```
