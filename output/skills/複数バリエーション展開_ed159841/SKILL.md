# 複数バリエーション展開

**説明**: ユーザーの要求に応じて複数の選択肢やパターンを生成するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 6 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 複数バリエーション展開

---

## テンプレートプロンプト

```
{対象物}について、{視点や条件}を考慮しつつ、{バリエーション数}パターンの{展開内容}を生成してください。各パターンは{具体性のレベル}で、{期待する効果}を狙ったものにしてください。また、{追加の要望}も反映してください。
```

## 変数一覧

- `{{対象物}}`: 展開の対象となるもの（例: 旅行ルート、システム構成、記事構成）
- `{{視点や条件}}`: 考慮すべき視点や制約条件（例: 個人の視点、時間制約、予算）
- `{{バリエーション数}}`: 生成するパターンの数（例: 3、5、複数）
- `{{展開内容}}`: 何を展開するか（例: ルート案、行動パターン、選択肢）
- `{{具体性のレベル}}`: どの程度詳細に書くか（例: 時間付きで具体的に、概要レベル）
- `{{期待する効果}}`: 読者や利用者に与えたい効果（例: ワクワクしながら選べる、全体像が把握できる）
- `{{追加の要望}}`: その他の特別な要求（例: 分岐の詳細、全く異なる趣向のパターン）

## 活用シーン

- 旅行プランの複数ルート提案
- システム設計の複数アーキテクチャ比較
- 記事構成の複数バリエーション作成

## 具体インスタンス

### インスタンス 1: 旅行ルート3パターン

> 元プロンプト: *記事いい感じですね　{旅行先}も少しは観光できるのかな？　あとルートabcみたいな３パターンで　もう少しいろいろ候補を含ませて　何をどう見るのか　読む側がワクワクしながら選ぶような感じで書いて欲しいです*

**変数の値**
  - `対象物`: {旅行先}
  - `視点や条件`: 観光も少しできるか
  - `バリエーション数`: 3
  - `展開内容`: ルート案
  - `具体性のレベル`: 時間付きで具体的に
  - `期待する効果`: 読む側がワクワクしながら選べる
  - `追加の要望`: 主要な分岐の詳細、{便名A}/{便名B}使用時の行動、ルートD/E/Fの追加

**実際のプロンプト**
```
{旅行先}について、観光も少しできるか考慮しつつ、{バリエーション数}パターンのルート案を生成してください。各パターンは時間付きで具体的に、読む側がワクワクしながら選ぶような感じで書いてください。また、ルートA・B・Cの主要な分岐でどうなるのかをより詳しく掘り下げるセクションも欲しいです。{便名A}、{便名B}を使う場合どういう行動になるのか時間とともに具体的に知りたい。ルートBに{便名B}つけるパターンをルートDとして、またこれまでと全く異なる趣向でルートEとFも考えてみて！
```

### インスタンス 2: システム挙動3視点

> 元プロンプト: *これで全体像をもういちど　個人からはどう見えるのか？　ハブはどう機能するのか？　{人数}人の社員が使うときの挙動で教えて*

**変数の値**
  - `対象物`: システム全体像
  - `視点や条件`: 個人から見えるか、ハブの機能
  - `バリエーション数`: 3
  - `展開内容`: 挙動パターン
  - `具体性のレベル`: 具体的な社員の使用例レベル
  - `期待する効果`: 全体像が把握できる
  - `追加の要望`: {人数}人の社員が使うときの挙動

**実際のプロンプト**
```
システム全体像について、個人からはどう見えるのか、ハブはどう機能するのかを考慮しつつ、3パターンの挙動パターンを生成してください。各パターンは具体的な社員の使用例レベルで、全体像が把握できることを狙ったものにしてください。また、{人数}人の社員が使うときの挙動で教えてください。
```
