# 記事企画相談テンプレート

**説明**: 記事作成の企画・構成・掲載を相談するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 6 件 | **ユニークユーザー**: 5 人 | **カテゴリ**: 記事作成の企画相談

## テンプレートプロンプト

```
{記事テーマ}について、{掲載先}に掲載する記事を作成したい。{記事の種類}として、{記事の長さ}程度の文字数で、{記事の構成要件}を満たすようにしてほしい。また、{追加要件}にも対応してください。
```

## 変数一覧

- `{{記事テーマ}}`: 記事の主題や内容（例: 開発記、調査結果）
- `{{掲載先}}`: 記事を掲載するメディアやサービス名（例: 社内ブログ、外部メディア）
- `{{記事の種類}}`: 時系列記事、調査記事、レポートなど（例: 開発記のような時系列記事）
- `{{記事の長さ}}`: 目標文字数（例: 8000文字、2000文字程度）
- `{{記事の構成要件}}`: 構成に関する条件（例: 論点を厳選、自然な文体、繰り返しを避ける）
- `{{追加要件}}`: その他の要望（例: RSS取得して深く読む、1日あたりの重要リンクを抽出）

## 活用シーン

- 開発ブログの時系列記事を企画する
- 調査結果をまとめたレポート記事を作成する
- RSSフィードから重要なリンクを抽出して記事化する

## 具体インスタンス

### インスタンス 1: 開発記時系列記事

> 元プロンプト: *このトレースの中身を参照しながら、開発記みたいな時系列記事ってつくれそう？Homupeに載せる用につくりたいんだよね　完結してるのでいうと、とくにないんだけども*

**変数の値**
  - `記事テーマ`: 指定したトレースを参照した開発記
  - `掲載先`: 指定した掲載先
  - `記事の種類`: 開発記のような時系列記事
  - `記事の長さ`: 特に指定なし
  - `記事の構成要件`: トレース内容を時系列で整理し、重要なポイントを抽出
  - `追加要件`: 完結しているものがない場合はその旨を考慮

**実際のプロンプト**
```
あなたが指定したトレースの中身を参照しながら、開発記のような時系列記事について、指定した掲載先に掲載する記事を作成したい。記事の種類として、完結している開発の時系列記事として、特に文字数制限はないが、記事の構成要件として、トレース内容を時系列で整理し、重要なポイントを抽出してほしい。また、追加要件として、完結しているものがない場合はその旨を考慮してください。
```

### インスタンス 2: 調査記事RSS深掘り

> 元プロンプト: *うーん　OOを調べて記事にしてHomupeに掲載して　とか　結構何回もやってるんですけど　これも会話サマリー作成ですかね？*

**変数の値**
  - `記事テーマ`: 指定したテーマを調べた調査結果
  - `掲載先`: 指定した掲載先
  - `記事の種類`: 数日分の記事をRSS取得して深く読み込む調査記事
  - `記事の長さ`: 特に指定なし
  - `記事の構成要件`: 1日あたりの重要リンクを厳選、繰り返しを排除
  - `追加要件`: 会話サマリー作成も検討

**実際のプロンプト**
```
指定したテーマを調べて記事にして指定した掲載先に掲載する記事を作成したい。記事の種類として、数日分の記事をRSS取得してより深く読み込む調査記事として、記事の長さは特に指定しないが、記事の構成要件として、1日あたりのリンクで重要なものはそんなにいっぱいないので、繰り返しのやつが多いと思うので、重要なものだけを厳選してほしい。また、追加要件として、会話サマリー作成も検討してください。
```

### インスタンス 3: 長文レポート改善

> 元プロンプト: *文字数が多いのはレポートの厚みとして重要なので、8000文字でもいいとおもう　ただし読んでてつまらない同じ構成の繰り返しをやめる　論点は厳選して、自然な文体で　一つの論点をもう少し深掘りして詳細に論じる*

**変数の値**
  - `記事テーマ`: 記事テーマ（具体的なテーマは元プロンプトにないため汎用）
  - `掲載先`: 指定した掲載先
  - `記事の種類`: レポート記事
  - `記事の長さ`: 8000文字
  - `記事の構成要件`: 同じ構成の繰り返しをやめる、論点を厳選、自然な文体、一つの論点を深掘り
  - `追加要件`: 文字数が多いのは厚みとして重要なので8000文字を維持

**実際のプロンプト**
```
記事テーマについて、指定した掲載先に掲載する記事を作成したい。記事の種類として、レポート記事として、記事の長さは8000文字程度で、記事の構成要件として、読んでてつまらない同じ構成の繰り返しをやめ、論点は厳選して、自然な文体で、一つの論点をもう少し深掘りして詳細に論じるようにしてほしい。また、追加要件として、文字数が多いのはレポートの厚みとして重要なので、8000文字を維持してください。
```
