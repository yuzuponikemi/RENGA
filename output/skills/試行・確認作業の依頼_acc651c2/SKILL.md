# 試行・確認作業の依頼

**説明**: 修正や実行の試行、動作確認を依頼するスキル

## テンプレートプロンプト

```
{アクション}して{対象}を{確認方法}しましょう
```

## 変数一覧

- `{{アクション}}`: 実行する操作（例: 修正、実行、コミット、追加）
- `{{対象}}`: 操作の対象（例: コード、目次、環境、スクリプト）
- `{{確認方法}}`: 確認の方法や目的（例: 動作確認、レビュー、正常動作の確認）

## 活用シーン

- 修正後に動作確認を依頼する
- 新機能のテスト実行を依頼する
- 設定変更後に確認を依頼する

  - `対象`: 先ほど失敗した対象

```
では先ほど失敗した対象について、再度実行してみましょう
```

### インスタンス 2: 仮想環境の修正

> 元プロンプト: *│ > ここの仮想環境にtorchvisionが足りて │ │   いないらしいので足して下さい　uv対応させたいのでproject.tomlにしましょう*

**変数の値**
  - `アクション`: 追加
  - `対象`: 仮想環境の指定したライブラリ
  - `確認方法`: 指定したツールに対応させたいので設定ファイルにしましょう

**実際のプロンプト**
```
ここの仮想環境に指定したライブラリが足りていないらしいので追加して下さい　指定したツールに対応させたいので設定ファイルにしましょう
```
