# 🔷 スキル定義: 調査・考察の要約

**説明**: 特定テーマの調査結果や考察を要約・整理するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 4 件 | **カテゴリ**: 調査・考察の要約

---

## テンプレートプロンプト（抽象）

```
{対象テーマ}について、{調査範囲}を基に{調査観点}を調査し、{出力形式}でまとめてください。特に{注目ポイント}に焦点を当て、{追加指示}を含めてください。
```

## 変数一覧

- `{{対象テーマ}}`: 調査・考察したい具体的なテーマ（例: 特定の機能の仕組み、ジョブの設定状況）
- `{{調査範囲}}`: 調査の対象範囲や情報源（例: 過去のログ、複数の資料、システム全体）
- `{{調査観点}}`: 調査の視点や目的（例: 原因の考察、現状の整理、詳細な調査）
- `{{出力形式}}`: 結果の出力方法（例: 日本語のドキュメント、箇条書き、レポート形式）
- `{{注目ポイント}}`: 特に重点的に調べる点（例: うまくいかない原因、現在の設定一覧）
- `{{追加指示}}`: その他の補足要件（例: 時系列で整理、図解を含める）

## 活用シーン

- システム障害の原因調査とレポート作成
- プロジェクト進捗の定期的なサマリー作成
- 複数資料からの知識統合とドキュメント化

# 🔶 具体インスタンス

### インスタンス 1: 問い生成の原因考察

> 元プロンプト: *なんでうまくいかないんでしょうか　{対象テーマ}に関して過去のログをまとめて、原因を考察して*

**変数の値**
  - `対象テーマ`: 問い生成
  - `調査範囲`: 過去のログ
  - `調査観点`: うまくいかない原因の考察
  - `出力形式`: 日本語のドキュメント
  - `注目ポイント`: うまくいかない原因
  - `追加指示`: 時系列で整理して原因を考察する

**実際のプロンプト**
```
{対象テーマ}に関して、{調査範囲}を基に{調査観点}を調査し、考察をまとめて{出力形式}にまとめてください。特に{注目ポイント}に焦点を当て、{追加指示}を含めてください。
```
