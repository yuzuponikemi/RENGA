# 調査指示の確認

**説明**: 調査作業の進捗や方法論を確認・議論するスキル
**ステータス**: ✅ 公開中 | **ソース数**: 5 件 | **ユニークユーザー**: 3 人 | **カテゴリ**: 調査指示の確認

---

## テンプレートプロンプト

```
{調査対象}について、{確認ポイント}を確認したい。{調査手法}の観点から、{懸念事項}について議論したい。{関連リソース}の管理状況も確認する。
```

## 変数一覧

- `{{調査対象}}`: 調査のテーマや対象（例: ウェブリサーチ、ブックリサーチ）
- `{{確認ポイント}}`: 具体的に確認したい内容（例: 見出しの取得方法、特定ツールの利用可否）
- `{{調査手法}}`: 使用する調査方法やツール（例: web_search、特定の検索機能）
- `{{懸念事項}}`: 議論したい問題点や限界（例: 結果の真正性、ツールの利用可否問題）
- `{{関連リソース}}`: 関連するファイルやフォルダ、環境設定（例: データフォルダ、環境変数）

## 活用シーン

- 調査プロジェクトの進捗確認ミーティングでの議論
- 新しい調査手法の導入検討時の意見交換
- 調査データ管理ルールの確認と合意形成

## トリガーフレーズ

## 具体インスタンス

### インスタンス 1: ウェブ調査の確認

> 元プロンプト: *うーん　つまり　web_research.headingsをあなたが提供したからうまくいったが、結局web_searchからこの見出しを見つけてきたわけではない？*

**変数の値**
  - `調査対象`: ウェブリサーチ
  - `確認ポイント`: 見出しの取得方法
  - `調査手法`: web_search
  - `懸念事項`: あなたが提供した見出しが実際に検索結果から取得されたものかどうか
  - `関連リソース`: （指定なし）

**実際のプロンプト**
```
ウェブリサーチについて、見出しの取得方法を確認したい。web_searchの観点から、あなたが提供した見出しが実際に検索結果から取得されたものかどうかについて議論したい。関連リソースの管理状況も確認する。
```
