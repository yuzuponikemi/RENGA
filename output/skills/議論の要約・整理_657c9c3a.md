# 🔷 スキル定義: 議論の要約・整理

**説明**: 複数の意見や情報を整理し、論点を明確にするスキル
**ステータス**: ✅ 公開中 | **ソース数**: 6 件 | **カテゴリ**: 議論の要約・整理

---

## テンプレートプロンプト（抽象）

```
{話題}について、{意見A}と{意見B}の間で{論点}が課題となっています。{意見A}の立場からは{主張A}が挙げられ、{意見B}の立場からは{主張B}が指摘されています。さらに{追加要素}を考慮すると、{結論方向性}という方向性が考えられます。ただし{懸念点}というリスクも存在するため、{次のアクション}が必要です。
```

## 変数一覧

- `{{話題}}`: 議論のテーマ（例: 特定技術の公開の是非）
- `{{意見A}}`: 一方の立場（例: 公開賛成派）
- `{{意見B}}`: もう一方の立場（例: 権利重視派）
- `{{論点}}`: 対立している核心的な問題（例: 知的財産権と共有のバランス）
- `{{主張A}}`: 意見Aの具体的な主張（例: コミュニティ貢献による評価向上）
- `{{主張B}}`: 意見Bの具体的な主張（例: 職務発明の保護）
- `{{追加要素}}`: 考慮すべき追加情報（例: 特定技術の活用）
- `{{結論方向性}}`: 導き出される方向性（例: 部分的公開と社内管理の併用）
- `{{懸念点}}`: 残るリスクや問題点（例: 悪用や抜け駆けの可能性）
- `{{次のアクション}}`: 次に取るべき行動（例: 法的整理とポリシー策定）

## 活用シーン

- 会議での複数意見を議事録として整理する
- オンラインディスカッションの論点をまとめる
- 技術選定における賛否両論を構造化する

**実際のプロンプト**
```
OSS公開の是非について、OSS公開賛成派と会社の権利重視派の間で知的財産権と共有のバランスが課題となっています。OSS公開賛成派の立場からはコミュニティ貢献による評価向上が挙げられ、会社の権利重視派の立場からは職務発明の保護が指摘されています。さらにブロックチェーン技術の活用を考慮すると、部分的公開と社内管理の併用という方向性が考えられます。ただし悪用や抜け駆けの可能性というリスクも存在するため、法的整理とポリシー策定が必要です。
```
