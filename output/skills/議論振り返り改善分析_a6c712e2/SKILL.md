# 議論振り返り改善分析

**説明**: 過去の議論やログを分析し、改善点や原因を特定するスキル
**ステータス**: 🔒 審査中 | **ソース数**: 3 件 | **ユニークユーザー**: 2 人 | **カテゴリ**: 議論の振り返りと改善

---

## テンプレートプロンプト

```
前回の議論を思い出して、{対象章}には何が書かれていたかを渡すという改善を実施してください。また、{未確認項目}が未確認ということは、{検出ツール}のループでも発見できなかったという事ですかね。{検出ツール}のログから、なぜ発見できなかったのか分析してください。
```

## 変数一覧

- `{{対象章}}`: 振り返り対象となる章やセクション名（例: 第3章、設計書）
- `{{未確認項目}}`: 未確認だった具体的な項目や問題（例: エラーケース、バグ）
- `{{検出ツール}}`: 使用した検出ツールや手法の名前（例: test-smith、ログ分析ツール）

## 活用シーン

- 過去の会議議事録を振り返り、未対応タスクを洗い出す
- テスト工程で見逃したバグの原因をログから分析する
- コードレビューで指摘された問題点を過去の議論と照合する

## トリガーフレーズ

- `改善点を洗い出して`
