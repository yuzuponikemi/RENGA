# 🔷 スキル定義: 非同期タスク進捗確認

**説明**: バックグラウンドタスクの進捗を確認し、状況に応じて再スケジュールや結果通知を行う
**ステータス**: ✅ 公開中 | **ソース数**: 7 件 | **カテゴリ**: 非同期タスク進捗確認

---

## テンプレートプロンプト（抽象）

```
Check the status of background task {task_id} ({task_name}). Use TaskOutput with block=false. {status_check_instruction}. If still running, schedule {wakeup_seconds}s wakeup. If pipeline finished or failed, read {output_file_path} for full details, summarize {result_summary_target}, and push notification to user.
```

## 変数一覧

- `{{task_id}}`: バックグラウンドタスクのID（例: {タスクID}）
- `{{task_name}}`: タスクの表示名（例: {プロジェクト名} {処理内容}）
- `{{status_check_instruction}}`: 進捗確認の具体的な指示（例: 現在のパス/ステップを報告。要約中なら{数値}秒後に再スケジュール、ファクトチェック中なら{数値}秒後に再スケジュール）
- `{{wakeup_seconds}}`: タスクがまだ実行中の時に再スケジュールする秒数（例: {数値}）
- `{{output_file_path}}`: 完了/失敗時に読み込む出力ファイルのフルパス（例: /tmp/{システム名}/tasks/{タスクID}.output）
- `{{result_summary_target}}`: 結果から抽出・要約する内容（例: スコアと主要コンテンツ、最終スコアと主要トピック）

## 活用シーン

- AIによる長文要約パイプラインの進捗確認と結果通知
- バッチ処理の品質チェックジョブの状態監視
- 複数ステップからなる非同期データ処理の進捗管理
